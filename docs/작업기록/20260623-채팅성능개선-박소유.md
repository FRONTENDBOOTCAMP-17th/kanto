## 1. 작업 개요

채팅 메시지 전송 시 발생하는 불필요한 클라이언트 사이드 DB 호출 제거 및 로딩 UX 개선.

---

## 2. 변경 사항

### 2-1. 메시지 전송 전 검증 로직 서버 이전 (`ChatRoomClient.tsx`, `actions.ts`)

**변경 전**

메시지 전송 버튼을 누를 때마다 클라이언트에서 서버 액션을 **2번 연속 호출**한 뒤 실제 메시지를 전송했다.

```ts
// ChatRoomClient.tsx — 전송 핸들러
if (await checkUserSuspendedAction(partner.id)) {  // DB 호출 1
  setSendError("정지된 사용자입니다.");
  return;
}
if (await checkBlockedAction(partner.id)) {        // DB 호출 2
  setSendError("차단된 사용자와는 메시지를 주고받을 수 없습니다.");
  return;
}
// 이후 sendMessageAction 호출                      // DB 호출 3+
```

왕복 네트워크 비용이 메시지 1건당 최소 3회 발생했고, 정지·차단 대상이 아닌 일반 전송에서도 매번 동일하게 소요됐다.

**변경 후**

`checkUserSuspendedAction`, `checkBlockedAction` 두 클라이언트 호출을 제거하고, 기존 `sendMessageAction` 내부에서 같은 검증을 수행하도록 이전했다.

```ts
// actions.ts — sendMessageAction 내부
if (partnerId !== null) {
  const { data: partnerData } = await supabase
    .from("users")
    .select("suspended_until")
    .eq("id", partnerId)
    .single();
  if (partnerData?.suspended_until && new Date(partnerData.suspended_until) > new Date()) {
    throw new Error("정지된 사용자입니다.");
  }

  if (await isBlockedPair(userData.id, partnerId)) {
    throw new Error("차단된 사용자와는 메시지를 주고받을 수 없습니다.");
  }
}
```

- 클라이언트→서버 왕복이 3회 → 1회로 줄었다.
- 검증 로직이 서버(Server Action) 안에서만 실행되므로 클라이언트에서 우회가 불가능하다.
- 에러는 기존과 동일하게 `catch` 블록에서 `sendError` 상태로 처리된다.

**수정 파일**
- `src/components/common/chat/chatPanel/room/ChatRoomClient.tsx` — `checkUserSuspendedAction`, `checkBlockedAction` import 및 호출 제거
- `src/components/common/chat/chatPanel/room/actions.ts` — `sendMessageAction` 내부에 정지·차단 검증 추가

### 2-2. 채팅방 로딩 Skeleton UI (`ChatRoom.tsx`, `skeleton.tsx`)

**변경 전**

채팅방 데이터를 불러오는 동안 "loading" 텍스트만 표시됐다.

```tsx
<div className="flex items-center justify-center h-full text-sm text-gray-400">
  {t("loading")}
</div>
```

**변경 후**

실제 채팅방 레이아웃(헤더 / 메시지 영역 / 입력창)과 같은 구조의 Skeleton으로 교체했다.

```tsx
<div className="flex flex-col h-full w-full">
  {/* 헤더: 아바타 + 이름 */}
  <div className="flex items-center gap-3 px-4 py-3 border-b ...">
    <Skeleton className="w-8 h-8 rounded-full" />
    <Skeleton className="w-32 h-4" />
  </div>
  {/* 메시지 영역: 좌우 번갈아 배치된 말풍선 */}
  <div className="flex-1 flex flex-col gap-3 px-4 py-4 ...">
    <Skeleton className="w-2/5 h-9 rounded-2xl self-start" />
    <Skeleton className="w-1/2 h-9 rounded-2xl self-end" />
    <Skeleton className="w-1/3 h-9 rounded-2xl self-start" />
    <Skeleton className="w-2/5 h-9 rounded-2xl self-end" />
    <Skeleton className="w-1/4 h-9 rounded-2xl self-start" />
  </div>
  {/* 입력창 */}
  <div className="px-4 py-3 border-t ...">
    <Skeleton className="w-full h-10 rounded-full" />
  </div>
</div>
```

레이아웃 시프트 없이 채팅방 UI 구조를 미리 보여주어 체감 로딩 시간이 줄어든다.

**신규 파일**
- `src/components/ui/skeleton.tsx` — shadcn/ui 스타일의 `Skeleton` 컴포넌트 (`animate-pulse` + `bg-muted`)

**수정 파일**
- `src/components/common/chat/chatPanel/room/ChatRoom.tsx` — 로딩 분기에 Skeleton UI 적용
