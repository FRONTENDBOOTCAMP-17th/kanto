# 구인구직 목록 카드(JobCard) 정보 보강

구인구직 목록의 카드가 **제목·회사명·급여·위치·좋아요**만 담고 있어 목록만 보고 판단할 정보가 부족했다. 목록에서 바로 가치가 높은 항목을 추가해 카드 정보 밀도를 높였다.

- **추가**: 공고일, 마감일(D-day), 고용형태 배지
- **제외**: 지원자 현황(지원 전용 테이블이 없고 "채팅=지원" 모델이라 별도 설계 필요), 모집인원(DB에 없는 새 필드 — 추후 별도 작업)

핵심: 추가한 3개 필드는 **이미 `getJobList`가 조회하는 데이터**라 DB 마이그레이션·서비스/쿼리 변경 없이 표시 레이어만 수정했다.

---

## 데이터 출처 (모두 기존 조회분 재사용)

| 표시 항목 | 데이터 | 위치 | 비고 |
|---|---|---|---|
| 공고일 | `post.created_at` | `posts` (Post) | 이미 select `*` |
| 마감일/D-day | `job.deadline` (`YYYY-MM-DD`, 필수) | `jobs` | 이미 select `jobs(*)` |
| 고용형태 | `job.employee_type` (정규직/계약직/파트타임) | `jobs` | 이미 select, 필터에도 사용 중 |

`getJobList`(`src/services/job/job.ts`)의 select가 이미 `*, jobs(*), users(...)`라 **변경하지 않음**.

# 2차 작업: 카드 레이아웃 개편 — 반응형 4단 (2026-06-17)

## 배경 / 문제
1차 작업으로 정보는 보강했으나, 페이지 컨테이너가 `max-w-7xl`(1280px)이라 **데스크탑처럼 넓은 화면에서 카드 내부가 비어 보이는** 문제가 있었다. 세로 스택 한 줄로는 넓은 폭을 채우지 못했다.

## 해결: 화면 폭에 따른 2개 레이아웃
- **md(768px) 미만**: 기존 세로 스택 그대로 유지 (모바일에서 4단은 너무 좁음)
- **md 이상**: 4단 가로 배치
  1. **회사** — 회사명(굵게)
  2. **본문** — 공고 제목 + 고용형태 배지 / 급여·위치 (좌측 구분선 `border-l px-4`)
  3. **우대사항** — `job.preferred` 2줄 클램프, 없으면 `"우대사항 없음"`(회색)
  4. **우측** — 찜 버튼 아래로 D-day 배지 + 등록 시간

`md:hidden`(모바일 블록) / `hidden md:flex`(데스크탑 블록)으로 분기.

## 데이터
- **우대사항**: `job.preferred`(`jobs.preferred`, `string | null`) — **이미 조회 중**이라 추가 쿼리·마이그레이션 없음. `JobList`에서 `preferred={job.preferred}`로 전달, `JobCardProps`에 `preferred: string | null` 추가.

## 주요 설계 결정
- **`LikeButton`은 한 번만 렌더**: 자체 `useState`(isLiked)라 두 블록에 각각 두면 찜 상태가 분리됨 → 우측 컬럼에 단일 배치하고, 데스크탑 전용 D-day·시간만 그 아래 `hidden md:flex`로 노출.
- **D-day 배지는 `dDayBadge` 변수로 1회 정의** 후 모바일/데스크탑 양쪽 재사용.
- **컬럼 폭을 비율(flex)로**: 처음엔 고정 폭(`w-40`/`w-48`/`w-24`)이라 창이 좁아질 때 본문만 줄어 텍스트가 무너졌다. 회사:본문:우대사항 = **`flex-2` : `flex-4` : `flex-3`** + 각 `min-w-0`로 바꿔 함께 유동적으로 축소(+ `truncate`/`line-clamp`로 넘침 방지). 우측 찜·마감 컬럼만 `md:w-24` 고정.

## 스타일 디테일
- **고용형태 배지 색 구분**(`employeeTypeClass`): 정규직=파랑(`blue-50/600`), 계약직=앰버(`amber-50/600`), 파트타임=초록(`green-50/600`), 그 외=회색.
- **D-day 배지 색 단순화**(`dDayClass`): `"오늘 마감"`만 빨강(`bg-red-50 text-red-400`), 나머지(D-n / 마감)는 회색(`bg-gray-100 text-gray-400`)으로 통일.
- **D-day 배지 아이콘 제거**: 기존 `CalendarClock` 아이콘 삭제(텍스트만), 미사용된 import 정리.
- **텍스트 크기**: 전반적으로 한 단계 상향. 단 데스크탑 블록은 **md 구간에서 한 단계 작게, `lg:` 이상에서 복귀**(예: 제목 `text-base lg:text-lg`, 회사·급여 `text-sm lg:text-base`, 배지·위치·우대사항·D-day `text-xs lg:text-sm`).

## 변경 파일
- `src/app/(user)/job/_components/JobCard.tsx` — props·레이아웃·스타일 전면 수정
- `src/app/(user)/job/_components/JobList.tsx` — `preferred` 전달

## 검증
- `npx tsc --noEmit` — 변경 파일 신규 타입 에러 없음(기존 무관 에러 11건은 `.next/types/*`·admin 등).
- 모바일(md 미만) 세로 스택 / md+ 4단 전환, 창 축소 시 컬럼 비율 유동 확인 필요(브라우저).

> ⚠️ 이 2차 섹션의 카드 컬럼 구성(우대사항 포함)은 **3차 작업에서 다시 변경**됨. 최종 상태는 아래 3차 참고.

# 3차 작업: 카드 최종 조정 + 구인 글쓰기/DB 확장 (2026-06-18)

## A. 카드(JobCard) 최종 조정
2차의 4단 구성을 사용자 피드백에 따라 재배치했다.

- **우대사항 컬럼 제거** (`preferred` prop·JobList 전달 정리) → 데스크탑은 **근무시간**을 새 컬럼으로 사용.
- **데스크탑(md+) 최종 4단**: 회사(`flex-2`, 굵게) / 제목·고용형태 배지(`flex-4`, 좌측 `border-l`, `self-stretch`로 구분선이 카드 전체 높이) / 급여·위치(`flex-3`, 세로 배치) / 근무시간(`flex-2`, 라벨 없이 값만).
- **우측 컬럼**: 찜 버튼(상단) + 오늘 마감 배지·등록시간(하단). `justify-between`으로 마감·시간을 카드 하단에 정렬.
- **모바일(md 미만)**: 제목·회사 / 근무시간(값만) / 급여·위치 순. 마감·등록시간은 우측 하단으로 이동(2차의 좌측 하단에서 변경).
- **급여 `₱` 기호 제거**(표시용 하드코딩 문자였음) → 숫자만.
- **D-day 색 추가 단순화**: `"오늘 마감"`만 빨강(`text-red-400`, 톤 완화), 그 외 회색.
- **근무시간 null 처리**: `work_hours`가 null(시간협의)이면 `JobList`에서 `"시간협의"` 문자열로 전달.

## B. DB 스키마 확장 (적용 완료)

```sql
  add column is_time_negotiable boolean not null default false,
  add column preferred_tags text[];
alter table public.jobs
  alter column work_hours drop not null;   -- 시간협의 시 null 저장
```

## C. 글쓰기 폼 — 근무 요일 + 시간 협의
- **근무 요일**: 월~일 개별 토글 + `평일/주말/매일` 프리셋. 상태 `workDays: string[]`.
- **시간 협의**: 체크박스(`is_time_negotiable`). 체크 시 근무시간 Select·요일 버튼 `disabled`, 제출 시 `work_hours`/`work_days`를 `null`로 저장.

- 상수: `WORK_DAYS`, `DAY_PRESETS` (`src/type/job/jobCreate.ts`).

## D. 우대 사항 — 카테고리 선택 모달
- 자유 텍스트 → **카테고리 다중선택 + 그 외 직접입력** 혼합.

# 4차 작업: 상세 페이지 표시·UI 다듬기 (2026-06-18)

- `JobTitle`: 아이콘(Eye) 옆 "조회수:" 텍스트 제거 → 아이콘 + 숫자만.

## 데스크탑 텍스트 크기 한 단계 상향
- 상세 전 컴포넌트(JobTitle/JobInfo/JobAuthorInfo/JobContent/CompanyInfo)에 `md:` 변형 추가(모바일은 기존 유지).
  - 예: 제목 `text-2xl md:text-3xl`, 섹션 제목 `text-base md:text-lg`, 본문 `text-sm md:text-base`, 라벨·태그 `text-xs md:text-sm`.

## 사진 크기
- `page.tsx`: 모바일은 전체 폭 유지, **데스크탑만 절반 폭 + 가운데 정렬**(`w-full md:w-1/2 mx-auto`) → 데스크탑에서 과하게 크던 이미지 축소.
