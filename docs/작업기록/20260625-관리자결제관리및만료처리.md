# 관리자 결제 관리 및 결제 만료 처리

## 작업 내용

### 1. 관리자 결제내역 관리 페이지 추가
- `AdminSidebar.tsx`: 사이드바에 "결제 관리" 항목 추가 (`/admin/payments`)
- `src/services/admin/adminTransactions.ts`: 관리자용 전체 거래 조회 서비스 추가
- `src/app/(admin)/admin/payments/page.tsx`: 서버 컴포넌트 페이지
- `src/app/(admin)/admin/payments/_components/AdminPaymentsClient.tsx`: 클라이언트 컴포넌트
  - 채팅 관리 UI 패턴 동일하게 적용 (헤더, 검색창, 테이블, 페이지네이션)
  - 상태별 필터 (전체 / 결제대기 / 결제완료 / 거래완료 / 취소됨 / 만료됨)
  - 완료 거래 합계 금액 표시
  - 검색: 게시글 · 구매자 · 판매자

### 2. 결제 상태 라벨 변경
- `paid` 상태: "에스크로" → "결제완료" (ko / en / fil 모두)
- 적용 파일: `messages/ko.json`, `en.json`, `fil.json`, `AdminPaymentsClient.tsx`, `ProfileTransactionsSection.tsx`

### 3. 결제 카드 만료 처리 (클라이언트 사이드)
- 문제: Xendit webhook은 배포 환경에서만 동작 → 로컬에서 24시간 지난 `pending` 건이 만료 처리 안 됨
- 해결: UI 레벨에서 `created_at + 24h < now` 체크로 만료 여부 판단
- `PaymentCard.tsx`
  - `isTimedOut` 계산 후 만료 시 expired UI 렌더링
  - `expired` / `cancelled` / `isTimedOut` 상태에서 카드 `opacity-60` + `pointer-events-none` 블록
- `MessageList.tsx`
  - `pending` 상태: 카드 하단에 회색 pill → "24시간 내에 결제하지 않으면 만료됩니다"
  - `expired` 또는 시간 초과: 회색 pill → "결제 기간이 만료되었습니다"
- i18n 키 추가: `pendingNotice`, `expiredNotice` (ko / en / fil)

### 4. 거래내역 프로필 섹션 (커밋 별도)
- `ProfileAside.tsx`: "거래내역" 탭 추가
- `ProfileCard.tsx`: `ProfileTransactionsSection` 렌더링 연결
- `ProfileTransactionsSection.tsx`: 스켈레톤 로딩 · 빈 상태 · 상태 뱃지 포함 거래 목록
- `profile/_lib/actions.ts`: `getMyTransactions()` 서버 액션 (RLS 적용)
