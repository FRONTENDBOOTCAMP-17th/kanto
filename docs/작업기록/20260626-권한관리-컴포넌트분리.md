# 권한 관리 페이지 컴포넌트 분리

**날짜:** 2026-06-26  
**작성자:** 김도혁  
**대상:** `src/app/(admin)/admin/operation/permissions/_components/`

---

## 배경

`PermissionsClient.tsx` 한 파일이 어드민 목록, 팀 목록, 어드민 추가 세 탭의 UI와 상태, 서버 액션 호출을 모두 담당하고 있었다. 894줄에 달하는 단일 파일로, 탭 하나를 수정할 때도 전체 파일을 읽어야 했다.

---

## 문제점

### 1. 단일 파일에 세 탭의 로직이 혼재

`PermissionsClient.tsx`가 아래를 모두 담당하고 있었다.

- **어드민 목록 탭**: 그룹별 접기/펼치기 테이블, 삭제·승격 확인 흐름
- **팀 목록 탭**: 팀 목록 테이블, 팀 상세(권한 토글·멤버 관리), 드래그 앤 드롭 배정, 미배정 어드민 영역
- **어드민 추가 탭**: 유저 검색 모달, 팀 검색 모달, 추가 폼

### 2. 상태가 한곳에 집중

세 탭이 각자 필요로 하는 로컬 상태가 모두 최상위 컴포넌트에 선언되어 있었다.

```ts
// 어드민 목록용
const [pendingDelAdminId, setPendingDelAdminId] = useState<number | null>(null);
const [pendingRoleChangeId, setPendingRoleChangeId] = useState<number | null>(null);
const [expandedGroups, setExpandedGroups] = useState<Set<string>>(new Set());

// 팀 목록용
const [selectedTeamDetail, setSelectedTeamDetail] = useState<Team | null>(null);
const [draggingAdminId, setDraggingAdminId] = useState<number | null>(null);
const [addingTeam, setAddingTeam] = useState(false);
const [pendingDelTeamId, setPendingDelTeamId] = useState<number | null>(null);

// 어드민 추가용
const [selectedUser, setSelectedUser] = useState<UserResult | null>(null);
const [selectedTeam, setSelectedTeam] = useState<Team | null>(null);
const [userModalOpen, setUserModalOpen] = useState(false);
```

### 3. `useTransition` 하나로 모든 비동기 처리

탭을 불문하고 모든 서버 액션이 단일 `useTransition`으로 처리되어 있어, 어드민 삭제 중에 팀 생성 버튼도 `disabled` 처리가 되는 문제가 있었다.

```ts
const [isPending, startTransition] = useTransition();

// 모든 액션이 같은 isPending을 공유
function handleDeleteAdmin(id: number) {
  startTransition(async () => { await revokeAdmin(id); ... });
}
function handleAddTeam() {
  startTransition(async () => { const newTeam = await createTeam(name); ... });
}
```

### 4. 모달의 상태 리셋을 `useEffect`로 처리

`UserSearchModal`과 `TeamSearchModal`이 `open` prop으로 표시를 제어하면서, 열릴 때마다 상태를 초기화하기 위해 `useEffect` 안에서 `setState`를 직접 호출하고 있었다.

```ts
useEffect(() => {
  if (open) { setQuery(""); setDebouncedQuery(""); ... }
}, [open]);
```

이로 인해 아래 경고가 발생했다.

```
Error: Calling setState synchronously within an effect can trigger cascading renders
```

## 해결 방법

### 단계 1 — 탭별 컴포넌트 분리

각 탭을 독립 파일로 분리하고, 해당 탭에서만 쓰이는 상태와 핸들러를 각 파일 안으로 이전했다.

```
PermissionsClient.tsx (894줄)
  ↓
PermissionsClient.tsx (90줄)   — admins·teams 공유 상태 + 탭 바만
AdminListTab.tsx               — 어드민 목록 탭
TeamsTab.tsx                   — 팀 목록 탭
CreateAdminTab.tsx             — 어드민 추가 탭
constants.ts                   — PERMISSIONS (두 탭에서 공유)
```

**분리 기준:** 부모가 자기 자신을 위해 쓰지 않는 상태는 자식에게 이전한다.

`admins`와 `teams`는 세 탭이 함께 읽고 수정하므로 `PermissionsClient`에 유지했다. 각 탭은 부모에서 콜백 props를 받아 공유 상태를 업데이트한다.

```tsx
// PermissionsClient.tsx — 탭 결과를 받아 공유 상태 갱신
<AdminListTab
  admins={admins}
  teams={teams}
  onAdminDeleted={(id) => setAdmins((prev) => prev.filter((a) => a.id !== id))}
  onAdminPromoted={(id) => setAdmins((prev) => prev.map((a) => a.id === id ? { ...a, role: "super_admin" } : a))}
/>
```

### 단계 2 — TanStack Query 도입

각 탭이 자체 `useMutation`을 갖도록 하여, 탭별 `isPending`이 분리되었다.

**변환 전후 비교:**

```ts
// 이전 — 모든 액션이 단일 useTransition 공유
const [isPending, startTransition] = useTransition();

function handleDeleteAdmin(id: number) {
  startTransition(async () => {
    await revokeAdmin(id);
    setAdmins((prev) => prev.filter((a) => a.id !== id));
    setPendingDelAdminId(null);
  });
}

// 이후 — 각 액션이 독립된 useMutation 보유
const deleteMutation = useMutation({
  mutationFn: revokeAdmin,
  onSuccess: (_, id) => { onAdminDeleted(id); setPendingDelAdminId(null); },
});

const promoteMutation = useMutation({
  mutationFn: (id: number) => setAdminRole(id, "super_admin"),
  onSuccess: (_, id) => { onAdminPromoted(id); setPendingRoleChangeId(null); },
});

const isPending = deleteMutation.isPending || promoteMutation.isPending;
```

유저 검색(`searchUsers`)에는 `useQuery`를 적용해 수동 디바운스 타이머와 `searching` 상태를 제거했다.

```ts
// 이전 — 타이머·상태 수동 관리
const timerRef = useRef<ReturnType<typeof setTimeout> | null>(null);
const [searching, setSearching] = useState(false);

function handleQueryChange(value: string) {
  if (timerRef.current) clearTimeout(timerRef.current);
  timerRef.current = setTimeout(async () => {
    setSearching(true);
    const data = await searchUsers(value);
    setResults(data);
    setSearching(false);
  }, 300);
}

// 이후 — useQuery + 디바운스 state
useEffect(() => {
  const timer = setTimeout(() => setDebouncedQuery(query), 300);
  return () => clearTimeout(timer);
}, [query]);

const { data: allResults = [], isFetching } = useQuery({
  queryKey: ["users", "search", debouncedQuery],
  queryFn: () => searchUsers(debouncedQuery),
  enabled: debouncedQuery.trim().length > 0,
  staleTime: 30_000,
});
```

### 단계 3 — 모달 상태 리셋: effect 제거 → 조건부 렌더링

`open` prop과 리셋 effect를 제거하고, 부모에서 조건부 렌더링으로 전환했다.

```tsx
// 이전 — open prop + effect로 상태 리셋
<UserSearchModal open={userModalOpen} ... />

// UserSearchModal 내부
useEffect(() => {
  if (open) { setQuery(""); setDebouncedQuery(""); }
}, [open]);

// 이후 — 조건부 렌더링으로 마운트/언마운트
{userModalOpen && (
  <UserSearchModal ... />
)}

// UserSearchModal 내부 — 마운트 시 포커스만
useEffect(() => {
  inputRef.current?.focus();
}, []);
```

모달이 닫힐 때 언마운트되므로 state가 자동으로 초기화된다. `open` prop과 리셋 effect 자체가 불필요해진다.

### 단계 4 — 모바일 반응형 개선

테이블이 모바일에서 가로로 넘치고 탭 바가 잘리는 문제를 수정했다.

| 위치 | 문제 | 적용 |
|---|---|---|
| 탭 바 | 3개 탭이 소형 화면에서 잘림 | `overflow-x-auto` 래퍼 추가 |
| 어드민 목록 테이블 | 4열이 모바일에서 가로 넘침 | `overflow-x-auto` + 추가일 `hidden md:table-cell` |
| 승격 확인 텍스트 | "슈퍼어드민으로 승격?" 긴 텍스트 | `hidden sm:inline` (모바일엔 확인 버튼만 표시) |
| 팀 목록 테이블 | 5열이 모바일에서 가로 넘침 | `overflow-x-auto` + 권한 `hidden lg:table-cell` + 생성일 `hidden md:table-cell` |
| 팀 멤버 테이블 | 추가일 컬럼이 모바일에서 좁음 | `overflow-x-auto` + 추가일 `hidden sm:table-cell` |

모바일(~767px)에서는 핵심 컬럼(계정, 권한, 팀 이름, 멤버 수, 액션)만 노출되고, 날짜·권한 목록 등 부가 정보는 태블릿 이상에서 순차적으로 표시된다.

### 단계 5 — UI 텍스트 및 버튼 개선

- **어드민 목록** 컬럼 헤더 `소속 팀 / 권한` → `권한`, 셀에 `whitespace-nowrap` 추가로 줄바꿈 방지
- **전체 권한** 배지 → **전체** (슈퍼어드민 행)
- **팀에서 제외** 텍스트 버튼 → X 아이콘 버튼으로 교체 (삭제 버튼과 동일한 스타일)

### 파일 구조

```

  _components/

    AdminListTab.tsx       (175줄)  ← 신규

```

### 상태 관리 구조

```
공유 상태 (admins, teams)  →  PermissionsClient (useState + 콜백 props)
탭별 로컬 상태             →  각 탭 컴포넌트 (useState)
서버 액션                  →  useMutation (탭별 독립 isPending)
유저 검색                  →  useQuery (30초 캐싱, 디바운스 300ms)
```

### 개선된 점

- 각 탭의 `isPending`이 분리되어 다른 탭 액션에 영향을 주지 않음
- `useEffect` 내 `setState` 경고 제거
- 모달 상태가 닫힐 때 자동 초기화 (조건부 렌더링)
- 유저 검색 결과 30초 캐싱 (같은 검색어 재요청 없음)
- `GroupHeader`, `AdminRow`가 내부 클로저에서 모듈 레벨 컴포넌트로 분리되어 React 리마운트 문제 해소
- 모바일에서 핵심 컬럼만 표시, 테이블 가로 스크롤 지원
- 긴 UI 텍스트 축약 및 텍스트 버튼 → 아이콘 버튼으로 공간 절약
