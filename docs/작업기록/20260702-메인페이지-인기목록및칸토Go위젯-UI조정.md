# 2026-07-02 메인페이지 인기 목록 및 칸토Go 위젯 UI 조정

메인페이지 인기 목록 UI 조정 + 검색-인기 목록 사이 칸토Go 미리보기 위젯 신설 세션 요약.

---

## 1. 인기 뱃지 위치 — 모바일에서 좌측 상단으로 이동
- **요청**: 메인페이지 인기 목록에서 모바일일 때 인기 뱃지가 이미지 기준 좌측 상단에 더 가깝게 붙었으면 함.
- **제약**: `PopularBadge`는 메인페이지(`MainCard`)뿐 아니라 usedgoods, rental 카드에서도 공용으로 쓰이는 컴포넌트라, 직접 수정하면 다른 페이지 뱃지 위치까지 바뀜.
- **수정**: `PopularBadge.tsx`에 `compact` prop 추가. `compact`일 때만 모바일 오프셋을 `top-1.5 left-1.5`(6px)에서 `top-0.5 left-0.5`(2px)로 줄이고, `md:` 이상에서는 기존 값(`top-1.5 left-1.5`) 유지. `MainCard.tsx`에서만 `<PopularBadge compact />`로 호출해 메인페이지 인기 목록에만 적용, usedgoods·rental은 영향 없음.
- **참고**: 뱃지는 `ContentCard.tsx`의 `absolute top-2 left-2` 래퍼 안에 다시 `absolute`로 얹히는 이중 구조라 실제 이동폭은 미묘함. 더 눈에 띄게 하려면 `top-0.5 left-0.5`를 `top-0 left-0`으로 조정 가능.

## 2. 인기 목록 카드 개수 — 모바일 4개 → 2개
- **요청**: 인기 목록(중고거래/구인구직/방렌트 각 섹션) 카드가 모바일에서도 4개씩 나오는데, 칸토고 콘텐츠가 더 들어가야 해서 메인페이지가 길어지는 걸 막고 모바일 성능도 개선하고 싶음.
- **구조 확인**: `PopularTabs.tsx`에서 `PopularList` 컴포넌트는 `md:hidden` 래퍼 안에서만 렌더링됨 → **모바일 전용 컴포넌트**. 데스크탑은 탭 UI로 별도 `grid-cols-4`를 직접 렌더링하므로 `PopularList`와 무관.
- **수정**: `PopularList.tsx`에서 `items.map(...)` → `items.slice(0, 2).map(...)`으로 변경해 모바일에서 카드 2개만 렌더링. CSS로 숨기는 대신 배열 자체를 잘라내 불필요한 DOM/이미지 렌더링을 애초에 발생시키지 않도록 함(성능 목적에 부합).
- **부수 정리**: 도달 불가능했던(`PopularList`가 항상 모바일에서만 렌더링되므로) 죽은 반응형 클래스 `md:grid md:grid-cols-4 md:gap-4`를 같은 라인 수정 범위 내에서 제거.
- **영향 없음**: 데스크탑 탭 패널(`PopularTabs.tsx`의 `activeItems.map`)은 그대로 4개 유지. `Popular.tsx`의 서버 fetch는 여전히 4개씩 slice(데스크탑용으로 필요).

## 3. 검색-인기 목록 사이 칸토Go 미리보기 위젯 신설
- **배경**: 배포 후 대대적 디자인 개편을 앞두고, 이미 구현되어 있지만 메인페이지 노출이 약한(Hero 캐러셀 4장 중 1장, 4초마다 로테이션) 칸토Go(위치 기반 번개모임) 기능을 검색-인기 목록 사이에 상시 노출하고 싶다는 요청.
- **아이디어 검토**: 최신 등록 매물 피드/후기 캐럿셀/카테고리 바로가기/환율 위젯 등 대안을 제시했으나, "칸토Go 관련 콘텐츠여야 한다"는 제약으로 좁혀 (1) 모임 카드 미리보기 (2) 정적 미니맵 (3) 모임 현황 통계 배너 (4) 주제 칩 바로가기 중 성능 부담이 가장 적은 **1번(카드 미리보기)** → 이후 **3번(통계 배너)** 순으로 채택.

### 3-1. `GoPreview.tsx` (신규) — 모임 카드 미리보기
- `getActiveMeetups()`(기존 서비스, `/go` 페이지와 공유) 결과를 `page.tsx`에서 한 번만 fetch해 prop으로 전달(중복 쿼리 방지).
- 최대 3개 카드: 주제 뱃지(`TopicBadge` 재사용) · 제목 · 위치 · 시간(`formatManilaTimeRange` 재사용), 클릭 시 `/go`로 이동(딥링크는 `GoClient`에 쿼리파라미터 지원이 없어 범위 밖으로 판단).
- 모바일 가로 스크롤 / 데스크탑 3열 그리드. 활성 모임 0개면 "아직 열린 번개모임이 없어요" CTA 배너로 폴백.
- 카드 텍스트는 인기 목록 `ContentCard` 컨벤션에 맞춤: 제목 `text-sm font-semibold`, 위치·시간 `text-xs text-gray-500` + `MapPin`/`Clock` 아이콘(w-3 h-3). 카드 배경 `bg-white`, 주제 뱃지 `bordered={false}`, 크기도 `px-2 py-0.5 text-[11px]`로 축소.
- "전체보기" 버튼은 이후 요청으로 제거(`tc`/`Common.viewAll` 미사용으로 정리).

### 3-2. `GoStats.tsx` (신규, `GoPreview` 내부 헤더 아래 배치) — 모임 현황 통계 배너
- "N개 모임 진행 중 · M명이 함께하고 있어요"(`Go.map.inProgress` 재사용 + 신규 `Main.goStats.participants`) + 존재하는 모든 카테고리 배지(`MEETUP_TOPICS`, "기타" 제외 6개) 한 줄 노출, 클릭 시 `/go` 이동.
- 배경은 흰 카드(`bg-white border border-gray-200`)로, 아이콘은 원형 틸 배지로 결정(초기 teal-500 배경 시안에서 변경).
- 카테고리 배지는 처음엔 개수 표기(`맛집/식사 2` 등) 포함했으나 "숫자 불필요, 존재하는 모든 카테고리를 보여달라" 요청으로 개수 제거 + 활성 모임 유무와 무관하게 `MEETUP_TOPICS` 전체(기타 제외) 상시 노출로 변경.
- **가로 마퀴 애니메이션**: 데스크탑은 배지를 2배 복제한 트랙을 `@keyframes marquee`(translateX 0 → -50%)로 무한 스크롤, 카드 hover 시 `group-hover:paused`로 정지, `motion-reduce:animate-none`으로 모션 감소 설정 존중, 좌우 끝은 `mask-image` 그라데이션으로 자연스럽게 페이드.
- **모바일 2행 레이아웃**: "세로 2줄로 보이되 애니메이션은 유지"라는 요청에 따라 카테고리를 절반씩 나눠 두 개의 독립된 마퀴 행으로 분리. 처음엔 `grid-rows-2 grid-flow-col`로 시도했으나 그리드 컬럼 폭이 위/아래 행에 동일하게 강제되는 문제 발견 → 각 행을 독립 `flex` 트랙으로 바꿔 배지가 자기 텍스트 폭만큼만 차지하도록 수정(위아래 정렬은 자연스럽게 어긋남).
  - 그 결과 행마다 텍스트 총량이 달라 체감 속도·반복 폭이 달라지는 부작용 발생 → 각 행의 텍스트 글자수에 비례해 `animation-duration`을 계산(CSS 변수 `--marquee-duration`으로 전달, `animate-[marquee_var(--marquee-duration)_linear_infinite]`)해 두 행의 픽셀/초 속도를 맞춤. 반복 횟수도 4배로 늘려(`ROW_REPEAT`) 텍스트가 적은 행이 컨테이너보다 좁아 "비었다가 채워지는" 현상을 방지.
  - 모바일 애니메이션 속도가 너무 빠르다는 피드백으로 `duration = setChars * 0.5`(기존 0.22), 최소값 `16초`(기존 8초)로 조정해 체감 속도를 절반 이하로 늦춤.
- 텍스트 크기: 모바일 `text-xs` / `text-[11px]`, 데스크탑 `text-base` / `text-sm`으로 분리("텍스트를 줄여달라"는 요청을 처음엔 문구 단축으로 잘못 이해해 수정했다가, "크기를 줄이라는 것"이라는 정정 이후 폰트 크기 반응형으로 재수정 — `Main.goStats.inProgressShort`/`participantsShort` 키는 이후 "모바일에선 14개 모임, 91명 참가중처럼 나오게 해달라"는 요청으로 다시 도입해 최종적으로는 모바일 전용 축약 문구 + 축소 폰트 두 가지를 함께 적용).
- 배지 색상은 한 차례 검정 배경(`bg-gray-900`)+흰 글씨로 바꿨다가 "이전이 낫다"는 피드백으로 `TopicBadge`의 주제별 색상으로 롤백.

### 3-3. 트러블슈팅 — 마퀴 애니메이션이 안 움직임
- `globals.css`에 `@keyframes marquee`를 추가했는데도 실제 배포된 CSS 번들에 해당 키프레임 정의가 누락되고 유틸리티 클래스(`.animate-[marquee_...]`)만 생성되는 현상 발견.
- 개발 서버 프로세스를 재시작해도 재현 → Turbopack이 디스크에 남긴 `.next` 빌드 캐시가 원인으로 판단.
- `.next` 폴더 삭제 후 `npm run dev` 재시작으로 해결(재생성 가능한 빌드 캐시라 삭제 안전). Playwright로 `document.styleSheets`의 `@keyframes` 목록과 `transform` 값 변화를 직접 확인해 검증.

## 5. 모임 카드 캐러셀 — 서버 페이지네이션 기반 무한 캐러셀
- **요청**: `GoPreview`의 모임 카드에 "무한 캐러셀"을 적용하고 싶음.
- **방향 확정**: 옵션 제시 후 "버튼/인디케이터 캐러셀(Hero 방식 재사용)"로 결정. 이후 세 차례 요구사항 정정을 거쳐 최종 형태를 확정:
  1. (1차, 폐기) 한 번에 카드 1장씩 전체 폭으로 전환되는 Hero형 단일 슬라이드.
  2. (2차, 폐기) 6개씩 페이지 단위로 그리드 전환("페이지" 개념).
  3. (최종) **페이지 개념 없이 1열로 카드가 나열되고, 버튼으로 한 칸씩 이동, 영역을 벗어나는 카드는 `overflow-hidden`으로 숨김**. 데이터는 서버에서 처음에 6개만 가져오고, 끝에 도달하면 추가로 6개씩 더 불러옴.
- **서비스 변경**: `src/services/go/go.ts`의 `getActiveMeetups()`에 선택적 `{ offset, limit }` 파라미터 추가(`.range()` 사용). 기존 호출부(`useLiveMeetups.ts`, `go/page.tsx`)는 인자 없이 호출하므로 하위호환 유지.
- **`GoPreview.tsx`**: 다시 `meetups.slice(0, 6)`으로 초기 6개만 `MeetupCarousel`에 전달(서버에서 한 번에 6개씩만 불러온다는 요구사항 반영).
- **`MeetupCarousel.tsx`(신규, 클라이언트 컴포넌트)**: `"use server"` 모듈인 `go.ts`의 `getActiveMeetups`를 클라이언트에서 직접 import해 Server Action으로 호출.
  - 카드 폭 고정(`w-64`=256px)+`gap-3`(12px)로 스텝 계산, `transform: translateX(-index*STEP)`로 한 칸씩 이동. 컨테이너는 `overflow-hidden` + 좌우 `mx-9` 여백을 둬서, 버튼이 카드 위에 겹쳐 텍스트를 가리던 문제를 해결(요청 반영: "버튼 크기만큼은 카드 영역이 차지하지 않도록").
  - **끊김 없는 로딩**: 처음엔 마지막 카드에서 "다음" 클릭한 시점에 fetch를 시작해 로딩 지연이 노출됐음("불러오는 기준이 이상하다, 컨텐츠가 끊기지 않는 것처럼 보여야 한다") → 마지막 카드 도달 **전**(`PREFETCH_THRESHOLD`장 남았을 때)에 `useEffect`로 미리 백그라운드 prefetch하도록 변경. 임계값은 2 → 3 → 4로 순차 조정(현재 4, 즉 6개 중 인덱스 2부터 다음 6개를 미리 불러옴).
  - 서버가 `limit`보다 적게 반환하면(더 이상 없음) `exhausted` 플래그 설정, 이후 마지막 카드에서 "다음"을 누르면 처음(index 0)으로 순환. 실측: 활성 모임 14개 기준 6 → (prefetch) 12 → (prefetch) 14에서 멈추고 이후 순환 확인.
  - **자동재생 제거**: 최초엔 Hero처럼 4초 자동재생 + hover 정지였으나, "시간이 지나면 자동으로 넘어가는 것은 없애달라" 요청으로 `setInterval`/`paused` 상태를 완전히 제거하고 버튼 클릭으로만 이동하도록 변경.
  - **하단 dot 인디케이터 제거**: "아래쪽 페이지가 보여지는 UI는 없애달라" 요청으로 페이지 인디케이터 UI(dot 목록)도 삭제. 좌우 버튼만 남음.
