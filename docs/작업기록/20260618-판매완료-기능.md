# 중고거래 판매완료 기능

## 개요
구매자가 안전결제에서 "수령 확인"을 누르면 해당 게시글이 자동으로 판매완료 처리됩니다.
목록 카드와 상세 페이지 이미지 전체에 "판매완료" 오버레이가 표시됩니다.

## DB 변경

```sql
-- supabase/migrations/20260618110000_add_is_sold_to_posts.sql
ALTER TABLE posts ADD COLUMN IF NOT EXISTS is_sold boolean NOT NULL DEFAULT false;
```

> Supabase 대시보드 SQL Editor에서 직접 실행 필요

## 동작 흐름

1. 구매자가 채팅 내 결제카드에서 **수령 확인** 클릭
2. `confirmReceiptAction()` 실행
   - `transactions.status = "released"`
   - `posts.is_sold = true`
   - `posts.is_reserved = false` (예약중 자동 해제)
3. 목록 페이지: 이미지 전체를 덮는 반투명 오버레이 + "판매완료" 텍스트
4. 상세 페이지: 동일한 오버레이 표시

## 상태 우선순위

| 상태 | 목록 표시 | 상세 표시 |
|---|---|---|
| `is_sold = true` | 전체 이미지 오버레이 "판매완료" | 전체 이미지 오버레이 "판매완료" |
| `is_reserved = true` | 좌상단 주황 배지 "예약중" | 전체 이미지 오버레이 "예약중" |
| 기본 | 없음 | 없음 |

`is_sold = true` 이면 `is_reserved`는 자동으로 `false`가 되므로 두 상태는 겹치지 않습니다.
