# 결제 흐름 변경 — Xendit Disbursement 연동

## 작업 개요

기존 에스크로 흐름에서 `released` 전환 시 판매자에게 실제 송금이 발생하지 않던 문제를 해결.  
Xendit Disbursement API를 연동해 구매자가 수령 확인 시 판매자 계좌로 자동 정산되도록 변경.

**기존 흐름**
```
구매자 수령확인 → status: released → is_sold: true  (판매자 송금 없음)
```

**변경 후 흐름**
```
구매자 수령확인
  → seller.bank_code / bank_account_number 조회
  → (없으면) 에러: "판매자가 정산 계좌를 등록하지 않았습니다"
  → status: released + is_sold: true
  → Xendit Disbursement API 호출 → 판매자 계좌로 정산
  → transactions.xendit_disbursement_id 저장
  → 시스템 메시지 발송
```

## 변경 파일 목록

### DB (Supabase 직접 적용)

```sql
ALTER TABLE users
  ADD COLUMN IF NOT EXISTS bank_code TEXT,
  ADD COLUMN IF NOT EXISTS bank_account_number TEXT,
  ADD COLUMN IF NOT EXISTS bank_account_name TEXT;

ALTER TABLE transactions
  ADD COLUMN IF NOT EXISTS xendit_disbursement_id TEXT;
```

적용 후 `npm run gen:types` 로 `src/type/supabase.ts` 재생성.

### `src/components/common/chat/chatPanel/room/paymentActions.ts`

- `confirmReceiptAction()` 수정:
  1. released 전환 전에 판매자 bank 정보 조회
  2. 계좌 미등록 시 에러 throw → PaymentCard 에러 영역에 표시
  3. `createDisbursement()` 호출
  4. `transactions.xendit_disbursement_id` 저장
  5. disbursement 실패 시 콘솔 에러만 기록 (released는 이미 완료됐으므로 롤백 없음)
