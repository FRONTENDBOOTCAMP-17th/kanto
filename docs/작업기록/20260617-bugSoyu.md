## 3. 지금까지 완성된 기능 / 작업

### 3-1. 네비게이션 히스토리 버그 수정
**문제**: 게시글 수정 후 "목록으로" 버튼 누르면 수정 페이지로 되돌아가는 현상  
**원인**: 수정 완료 후 `router.push()` 로 상세 페이지 이동 → 히스토리에 수정 페이지가 쌓임  
**해결**: `router.replace()` 로 교체 (히스토리에 수정 페이지 제거)

- `src/app/(user)/rental/create/_components/RentalCreateForm.tsx`
- `src/app/(user)/usedgoods/create/_components/CreateUsedGoodsForm.tsx`
- `src/hooks/useCreateJobForm.ts`

그리고 상세 페이지 "목록으로" 버튼은 `router.back()` → `router.push("/usedgoods")` 로 변경 (뒤로가기 대신 명시적 목록 이동).

### 3-2. DB 보안 — 민감 컬럼 노출 제거
**문제**: `users(*)` JOIN 쿼리가 `email`, `provider`, `updated_at` 등 민감 정보를 클라이언트에 노출  
**해결**: 모든 상세 페이지 쿼리에서 필요한 컬럼만 명시적으로 선택

변경 대상:
- `src/services/rental/rental.ts` — RENTAL_DETAIL_SELECT
- `src/services/job/jobDetail.ts` — JOB_DETAIL_SELECT
- `src/services/usedGoods/usedGoods.ts`
- `src/app/(user)/usedgoods/[id]/page.tsx`

허용 컬럼: `id, name, avatar_url, auth_id, role, post_count, created_at`

### 3-3. 작성자 null 방어 처리
**문제**: `posts.users` 가 null 일 수 있는데 `.users.x` 직접 접근 → 런타임 에러  
**해결**: 타입을 `RentSellerInfo | null`, `JobSellerInfo | null` 로 변경하고 전체 optional chaining(`?.`) 적용

### 3-4. DB — `notify_on_like` SECURITY DEFINER 추가
**문제**: `common_likes` INSERT 시 403 에러 발생  
**원인**: `notify_on_like()` 트리거 함수가 SECURITY DEFINER 없이 현재 유저 권한으로 실행 → notifications 테이블 쓰기 불가  
**해결**: DB 마이그레이션으로 `notify_on_like()`, `notify_on_like_delete()` 에 `SECURITY DEFINER` 추가

---

### 3-5. 찜(like) 카운트 버그 수정 — DB + 코드

#### DB 수정 (마이그레이션 `fix_post_like_count_trigger` 적용됨)
1. **중복 트리거 제거**: `trg_like_count`와 `trg_update_like_count` 둘 다 `update_like_count()` 호출 → `users.like_count` 2배 증가 버그. `trg_like_count` 삭제.
2. **`posts.like_count` 업데이트 트리거 신규 생성**: 기존 `update_like_count()`는 `users.like_count`(이 사람이 찜한 수)만 업데이트하고 `posts.like_count`(이 글이 받은 찜 수)는 업데이트 안 했음. → `update_post_like_count()` 함수 + `trg_post_like_count` 트리거 생성.
3. **기존 데이터 재계산**: 모든 `posts.like_count` 를 `common_likes` 실제 데이터 기준으로 재계산.

#### 코드 수정
- **`LikeButton.tsx`**: `onLikeChange?(liked: boolean)` 콜백 prop 추가 → 클릭 시 부모에 알림, 에러 시 rollback
- **`ContentCard.tsx`**: 로컬 `count` state 추가, `LikeButton`에 `onLikeChange` 연결 → 카드에서 하트 클릭 시 숫자 실시간 반영
- **`JobTitle.tsx`**: 로컬 `likeCount` state + 조회수 옆에 `<Heart /> {likeCount}` 표시, `InteractionButtons`에 `onLikeChange` 연결
- **`PostInfo.tsx`** (방렌트): 동일 패턴 적용
- **`UsedGoodsDetail.tsx`**: unlike 시 `prev - 1` → `Math.max(prev - 1, 0)` 으로 -1 방지

### 찜(Like) 시스템 구조
```
common_likes 테이블
  └── target_type: "post" | ...
  └── target_id: post.id

트리거:
  - trg_update_like_count  → update_like_count()     → users.like_count +/-1 (유저가 찜한 개수)
  - trg_post_like_count    → update_post_like_count() → posts.like_count +/-1 (글이 받은 찜 수) ← 이번 세션 추가
  - trg_notify_like        → notify_on_like()         → notifications 생성 (SECURITY DEFINER)
  - trg_notify_like_delete → notify_on_like_delete()  → notifications 삭제 (SECURITY DEFINER)
```

프론트 패턴:
- 상세 페이지: `useState(posts.like_count ?? 0)` + `onLikeChange` 콜백으로 optimistic update
- 카드: `ContentCard` 내부 `count` state + `LikeButton`의 `onLikeChange` 연결

### 이미지 없는 상세 페이지 레이아웃 원칙
- **구인구직**: 항상 `[JobInfo | JobAuthorInfo]` 2열 (이미지는 카드 내부에 옵션으로 따로)
- **중고거래**: 이미지 있으면 `[이미지 | 상품정보+판매자]`, 없으면 `[상품정보 | 판매자]` 2열
- **방렌트**: 이미지 있으면 `[이미지 | 숙소정보+집주인]`, 없으면 `[숙소정보 | 집주인]` 2열
