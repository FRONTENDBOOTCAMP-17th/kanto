## 1. WebVitalsReporter 컴포넌트

**파일**: `src/components/common/WebVitalsReporter.tsx`

### 뭐 하는 컴포넌트?
개발 중 브라우저 콘솔에서 Core Web Vitals 수치를 실시간으로 볼 수 있게 해줍니다.  
`development` 환경에서만 동작하고, `production`에서는 아무것도 하지 않습니다.  
(운영 환경의 CWV는 Sentry가 자동으로 수집합니다.)

### 사용법
별도 설정 없이 `app/layout.tsx`에 이미 추가되어 있습니다.  
로컬에서 `npm run dev` 후 페이지를 열면 콘솔에 아래와 같이 출력됩니다:

```
[CWV] LCP        1234ms  ✅ good
[CWV] CLS       0.023    ✅ good
[CWV] INP        180ms   ✅ good
[CWV] FCP        850ms   ✅ good
[CWV] TTFB       320ms   ✅ good
```

등급 기준:
- `✅ good` — Google 권장 기준 충족
- `⚠️ needs improvement` — 개선 필요
- `❌ poor` — 사용자 경험에 영향

### 왜 Next.js의 useReportWebVitals를 썼나?
`web-vitals` 라이브러리를 직접 설치하지 않아도 됩니다.  
Next.js에 내장된 `useReportWebVitals` 훅을 사용하면 별도 패키지 없이 동일하게 동작합니다.

## 2. 번들 분석기 (Bundle Analyzer)

**패키지**: `@next/bundle-analyzer` (devDependency로 설치됨)  
**수정 파일**: `next.config.ts`, `package.json`

### 뭐 하는 도구?
빌드 결과물의 JS 번들을 시각적으로 분석하는 도구입니다.  
어떤 라이브러리가 번들을 크게 만드는지 한눈에 볼 수 있습니다.

### 사용법
```bash
npm run analyze
```
빌드 완료 후 브라우저에서 시각화 페이지가 자동으로 열립니다.

- **클라이언트 번들**: 브라우저에서 실행되는 JS
- **서버 번들**: 서버 컴포넌트, API Route에 해당하는 JS

### 언제 쓰나?
- 무거운 라이브러리를 새로 추가했을 때
- 특정 페이지 로드가 느리다고 느껴질 때
- 배포 전 번들 크기 점검 시

> 일반 `npm run build`보다 느리므로, 로컬에서 필요할 때만 실행합니다.

### next.config.ts 변경 내용
```ts
// 추가된 부분
import bundleAnalyzer from "@next/bundle-analyzer";

const withBundleAnalyzer = bundleAnalyzer({
  enabled: process.env.ANALYZE === "true",  // ANALYZE=true 일 때만 활성화
});

// 기존 export에 withBundleAnalyzer 래핑 추가
export default withBundleAnalyzer(sentryConfig as NextConfig);
```
`ANALYZE` 환경변수가 없을 때는 완전히 비활성화되어 일반 빌드에 영향 없습니다.

## 3. 운영 환경 CWV 모니터링 (기존 Sentry 활용)

신규 코드 추가 없이, 기존 Sentry 설정(`tracesSampleRate: 1`)으로 이미 수집 중입니다.

**확인 방법**: Sentry 대시보드 → Performance → Web Vitals 탭

팀원이 별도로 설정할 것은 없습니다.
