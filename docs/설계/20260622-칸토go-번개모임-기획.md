# 칸토 go! — 번개모임 기능 기획서

> 작성일: 2026-06-22 · 상태: 기획 확정(구현 전)

## Context (배경 및 목적)

기존 칸토 서비스는 중고거래(usedgoods)·구인구직(job)·렌탈(rental) 중심의 생활 플랫폼이다.
여기에 **위치 기반 번개모임(일회성 오프라인 모임)** 기능 "칸토 go!"를 추가해, 사용자가
지도 위에서 주변에서 열리는 모임을 한눈에 보고, 핀을 눌러 참여(Join)하고, 참여자끼리
소통(단체채팅, v2)하도록 한다. 모임은 일회성이므로 끝나면 지도에서 사라지고 관련 데이터도
정리된다.

- **타겟 환경**: 앱이 필리핀/타갈로그(`fil`) + Xendit 결제를 쓰는 점으로 보아 필리핀 대상.
  → 지도는 **Google Maps** 사용(커버리지·reverse geocoding 우수).
- **목표**: 가벼운 일회성 오프라인 모임을 지도 UX로 직관적으로 개설·발견·참여하게 한다.

## 핵심 결정사항 (논의 확정)

| 항목 | 결정 |
|---|---|
| 지도 SDK | **Google Maps** (reverse geocoding으로 핀→주소/상호명 자동표시) |
| 위치 입력 | 지도 **핀 클릭 → 자동 주소·상호/건물명 표시 → 주최자가 상세 한 줄 보정** |
| 모임 정보 표시 | **확장형 bottom sheet(토스트) 하나로 완결** — 별도 상세페이지 없음 |
| 주제(topic) | **고정 카테고리 목록** (핀 색·아이콘 구분 및 필터링) |
| 참여 취소 | **모임 시작 전까지 자유 취소**(자리 반환), 시작 후 불가 |
| 안전/신고 | **기존 신고 인프라(`common_reports` + ReportModal) 재사용** |
| 모임 종료 | **종료시각 자동만료** → 핀 즉시 사라짐(쿼리 시점 필터링) |
| 단체채팅 | **단계적 분리** — v1 제외, **v2에서 그룹채팅** 구현 |
| 채팅 수명 | (v2) 종료 후 **24시간 유예 → 자동 완전삭제** |

## 사용자 플로우

```
[칸토 go! 진입]
   ↓ nav 아래로 큰 지도 표시, 진행 중인 모임이 주제별 색 핀으로 표시
[핀 클릭]
   ↓ 화면 하단에 직사각형 bottom sheet 등장
   ├─ 요약: 제목 · 주제 · 날짜/시간 · 장소(자동주소+상세)
   ├─ 주최자 프로필(좌) | 참여자 프로필 동그라미들(우)
   ├─ (시트를 위로 드래그하면 한줄설명·정원 현황 등 더 보임)
   └─ [Join] 버튼
[Join 클릭]
   ↓ 참여자로 등록, 정원 차감, 주최자에게 알림
   └─ (v2) 참여자 단체채팅방 입장
[모임 종료시각 도달]
   ↓ 핀이 지도에서 사라짐
   └─ (v2) 채팅방 24h 유지 후 자동 삭제
```

- 참여자 동그라미 **탭 시**: 별도 페이지 대신 **작은 프로필 팝오버/모달**(이름·아바타·지역).
- 모임 **개설**: 우측 하단 FAB 또는 상단 버튼 → 생성 폼.

## 화면 / UI 구성

### 1. 지도 화면 (`/go`)
- nav 바로 아래 전체 영역에 Google Map. 현재 위치 기준 초기 중심.
- 진행 중 모임(= `end_at > now()`)만 핀으로 렌더. 주제별 핀 색/아이콘 구분.
- 핀 클릭 → bottom sheet 오픈.

### 2. Bottom Sheet (토스트) — **신규 컴포넌트**
- 기존 모달 패턴 변형: `fixed inset-0` 오버레이 + `items-end` + `rounded-t-2xl bg-white`.
  참고: `src/components/common/ConfirmModal.tsx`(포털·오버레이 패턴), `src/components/common/Toast.tsx`.
- 확장형(드래그-업): 기본은 요약, 위로 끌면 한줄설명·정원 현황 노출.
- 프로필 동그라미: 기존 아바타 패턴 재사용
  (`src/app/(user)/profile/_components/profileAvatar.tsx`, `ChatListItem`의 원형+이니셜 fallback).

### 3. 모임 생성 폼 (`/go/create`)
- 기존 생성 폼 패턴 미러링: `src/hooks/useCreateJobForm.ts`(상태/검증/DB insert), `CreateJobForm.tsx`.
- **단일 스텝**으로 충분(번개모임 정보가 가벼움).

**생성 양식 필드**
| 필드 | 입력 방식 | 비고 |
|---|---|---|
| 제목 | 텍스트(필수) | `posts.title` |
| 주제 | 고정 카테고리 select(필수) | 핀 색/필터 |
| 날짜 | date(필수) | |
| 시작 시간 | time(필수) | `start_at` |
| **종료 시간** | time(필수) | `end_at` — **원안에 없었으나 자동만료 위해 추가** |
| 장소 | 지도 핀 클릭 → 자동주소 표시 | `lat/lng` + `address` |
| 상세 위치 | 텍스트 한 줄(선택) | "OO카페 2층" 등 보정 |
| 한줄 설명 | 텍스트(필수) | |
| 인원수 제한 | number(필수) | `max_participants` |

> **주제 고정 카테고리(초안, 추후 조정 가능)**: 맛집/식사 · 운동/스포츠 · 스터디/언어교환 ·
> 취미/문화 · 나들이/여행 · 친목/수다 · 기타

## 데이터 모델 (Supabase)

기존 패턴(`posts` 본체 + 타입별 확장 테이블, `post_type` 구분자) 그대로 따른다.
참고: `src/type/supabase.ts`, `src/services/job/job.ts`.

### `posts` (기존 재사용)
- `post_type = "meetup"` 신규 값, `title`, `user_id`(= 주최자), `status`.

### `meetups` (신규 — post_id로 1:1 조인)
```
post_id          FK posts(id)
topic            text         -- 고정 카테고리 키
start_at         timestamptz
end_at           timestamptz  -- 자동만료/핀 사라짐 기준
location_lat     double precision
location_lng     double precision
location_address text         -- reverse geocoding 결과
location_detail  text         -- 주최자 보정 한 줄 (nullable)
description      text         -- 한줄 설명
max_participants int
```

### `meetup_participants` (신규 — 다대다 참여 관계)
```
id               PK
meetup_post_id   FK posts(id)
user_id          FK users(id)
joined_at        timestamptz
status           text   -- "joined" | "cancelled"
unique(meetup_post_id, user_id)
```
- 주최자는 `posts.user_id`로 식별(참여자 행에 포함하지 않음).
- 현재 인원 = `joined` 참여자 수 + 1(주최자). 정원 초과 시 Join 비활성.

### (v2) 단체채팅 테이블 (신규)
기존 채팅은 **1:1 전용**(`chats.user_id_1/2`)이라 그대로 못 씀 → 그룹용 신규 테이블 필요.
```
meetup_chat_rooms:    id, meetup_post_id, created_at, expires_at(=end_at+24h), status
meetup_chat_messages: id, room_id, sender_id, content, type, created_at
```
- Realtime은 기존 패턴 재사용: `src/hooks/chat/useChatRoomRealtime.ts`의
  `supabase.channel()` + `postgres_changes` 구독 방식을 그룹방용으로 변형.

## 기능 상세

### 참여(Join) / 취소
- Join: `meetup_participants` insert(`status=joined`), 정원 검사 → 초과 시 버튼 비활성.
- 취소: **시작 전까지만** `status=cancelled`(또는 row 삭제)로 자리 반환. 시작 후 불가.
- 노쇼 패널티/평점: **추후**(v1 제외).

### 모임 종료 (자동만료)
- **핀 사라짐**: 쿼리 시점에 `end_at > now()` 필터 → 별도 크론 불필요.
- (v2) **채팅 삭제**: `end_at + 24h` 경과 시 방·메시지 삭제.
  구현 옵션: Supabase **pg_cron** 스케줄 잡 또는 접근 시 lazy cleanup.

### 알림 (재사용)
- `meetup_join`: 참여 발생 시 주최자에게(`receiver_id`, `related_type="meetup"`, `related_id=post_id`).
- `meetup_soon`(선택): 시작 N분 전 참여자 리마인드.
- 기존 `common_notifications` insert 한 줄로 처리.

### 안전/신고
- bottom sheet 및 프로필 팝오버에 신고 진입 → `ReportModal` 재사용("모임 신고/유저 신고").
- (선택) 첫 모임 참여 시 안전 안내 문구 노출 — **추후 결정**.
- 본인인증 강제: **추후**(가입 장벽 우려).

## 검증 (Verification)

**환경 준비**: `.env.local`에 `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY` 설정(Maps JS + Geocoding 활성화).

**Phase 1 수동 검증 시나리오** (`npm run dev` → `/go`):
1. 지도가 nav 아래 전체에 렌더되는지.
2. 모임 생성 → 폼에서 지도 핀 클릭 시 자동 주소/상호명 표시, 상세 한 줄 입력.
3. 저장 후 지도에 주제 색 핀 등장.
4. 핀 클릭 → bottom sheet에 제목·주제·날짜/시간·장소·주최자/참여자·Join 노출.
5. 다른 계정으로 Join → 참여자 프로필 추가, 주최자 알림 수신, 정원 도달 시 Join 비활성.
6. 시작 전 취소 → 자리 반환. 종료시각 지난 모임 → 핀 사라짐.
7. 신고 모달 동작.

**타입/빌드**: `npm run gen:types` 후 `npm run build` 통과 확인.
**테스트**: 프로젝트에 단위 테스트 관례가 약하므로 위 수동 시나리오를 1차 검증 기준으로 한다.
