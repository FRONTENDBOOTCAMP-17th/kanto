## 1. 배경 (왜 이 정리본을 만들었나)

칸토 go!(위치 기반 일회성 번개모임) 기능에 대해 **세 종류의 문서가 따로 존재**했고 서로 어긋나 있었다.

- **기획자 문서** — 확정 기획(구현 전). bottom sheet / 별도 생성 페이지 / 어드민 없음 / Realtime은 v2.
- **디자이너 종합 문서** — 디자인 완료 + **실제 구현 코드(nextjs-output)와 HTML 프로토타입 보유**. 우측 슬라이드 패널 / 생성 모달 / 어드민 포함 / v1부터 Realtime.
- **디자이너 코드 패키지 README** — `nextjs-output/` 적용 방법, Supabase SQL, 패키지 설치 안내.

디자이너와 기획자가 충분히 합의하지 못한 채 각자 진행해 **UI/범위/Realtime에서 충돌**이 있었다. 본 문서는 이 충돌을 사용자 결정으로 해소하고, 실제 코드베이스 사실에 맞춰 **단일 진실원천(single source of truth)**을 확정한다.

## 4. Google Maps API 불러오는 방법

### 4-1. Google Cloud Console 준비 (1회성)
1. 프로젝트 생성 → **결제 계정 연결**(필수, 무료 크레딧 범위 내).
2. **API 사용 설정**: `Maps JavaScript API` + `Geocoding API`.
3. **사용자 인증 정보 → API 키 생성**. 보안 제한 권장:
   - 애플리케이션 제한: **HTTP 리퍼러**(`localhost:3000/*`, 배포 도메인).
   - API 제한: 위 2개 API로만 한정.
4. **Map ID 발급**: Google Maps Platform → *Map Styles* → 신규 Map ID(JS). `AdvancedMarker`(핀)는 `mapId`가 **필수**다.

### 4-2. 패키지 & 환경변수
```bash
npm install @vis.gl/react-google-maps
```
```env
# .env.local  (git에 절대 커밋 금지 — .gitignore에 .env.local 포함 확인)
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=발급받은_키
NEXT_PUBLIC_GOOGLE_MAPS_MAP_ID=발급받은_MapID
```

### 4-3. 코드에서 로드 (핵심 패턴)
```tsx
// 최상위: APIProvider로 감싼다 (페이지 또는 go 레이아웃)
import { APIProvider, Map, AdvancedMarker } from "@vis.gl/react-google-maps";

<APIProvider apiKey={process.env.NEXT_PUBLIC_GOOGLE_MAPS_API_KEY!}>
  <Map
    mapId={process.env.NEXT_PUBLIC_GOOGLE_MAPS_MAP_ID}
    defaultCenter={{ lat: 14.55, lng: 121.05 }}  // 메트로 마닐라/BGC 초기값
    defaultZoom={13}
    gestureHandling="greedy"   // 한 손가락 패닝/핀치 줌
    zoomControl={true}         // ★ 줌인/줌아웃 버튼 (사용자 요구) — 메인 지도
    disableDefaultUI={false}
  >
    {meetups.map((m) => <AdvancedMarker key={m.postId} position={{lat,lng}} />)}
  </Map>
</APIProvider>
```
- **줌 요구사항**: 메인 지도와 **생성 모달의 미니지도 둘 다** `zoomControl={true}` + `gestureHandling="greedy"` 적용. 미니지도가 작아 핀 찍기 어려운 문제를 줌으로 해소.
- **Reverse Geocoding** (핀 클릭 → 주소): `useMapsLibrary("geocoding")` 로 `Geocoder` 인스턴스 생성 후 `geocoder.geocode({ location: {lat,lng} })` → `results[0].formatted_address`. → `useLocationPicker.ts` 에 캡슐화.
- `'use client'` 컴포넌트에서만 사용. 메인/모달 지도는 클라이언트 컴포넌트.

## 6. 데이터 모델 (Supabase 마이그레이션 — 보정본)

`post_type` 은 text이므로 `ALTER TYPE` 없음. FK는 `public.users` 기준.

```sql
-- posts 재사용: post_type = 'meetup', user_id = 주최자

CREATE TABLE meetups (
  post_id          UUID PRIMARY KEY REFERENCES posts(id) ON DELETE CASCADE,
  topic            TEXT NOT NULL,            -- meetupTopics 키
  start_at         TIMESTAMPTZ NOT NULL,
  end_at           TIMESTAMPTZ NOT NULL,     -- 자동만료/핀 사라짐 기준
  location_lat     DOUBLE PRECISION NOT NULL,
  location_lng     DOUBLE PRECISION NOT NULL,
  location_address TEXT NOT NULL,            -- reverse geocoding 결과
  location_detail  TEXT,                     -- 주최자 보정 한 줄 (nullable)
  description      TEXT NOT NULL,
  max_participants INT NOT NULL DEFAULT 6
);

CREATE TABLE meetup_participants (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  meetup_post_id UUID NOT NULL REFERENCES posts(id) ON DELETE CASCADE,
  user_id        UUID NOT NULL REFERENCES public.users(id),   -- ★ auth.users → public.users
  joined_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  status         TEXT NOT NULL DEFAULT 'joined' CHECK (status IN ('joined','cancelled')),
  UNIQUE(meetup_post_id, user_id)
);

-- v1 Realtime
ALTER PUBLICATION supabase_realtime ADD TABLE meetups;
ALTER PUBLICATION supabase_realtime ADD TABLE meetup_participants;
```
- 현재 인원 = `joined` 참여자 수 + 1(주최자). 정원 도달 시 Join 비활성, **88%↑ "마감임박" 뱃지 / 100% "마감"**.
- (v2) `meetup_chat_rooms(id, meetup_post_id, expires_at=end_at+24h, status)`, `meetup_chat_messages(...)`.

## 8. 단계별 작업 순서

### 사전 (블로킹)

   → verify: `src/components/go/*`, `src/hooks/go/*` 파일 존재 + 빌드 임포트 해결.

### Phase 1 (v1) — 지도 발견·참여 + 어드민
1. Google Maps 셋업(키·MapID·`APIProvider`) + `/go` 지도 페이지(메인 지도, **줌 컨트롤**, 현재 위치 중심) → verify: 지도 렌더·줌 동작.

3. `routes.ts` `go` 키 + `Header` `NAV_ITEMS` + i18n 3개 언어 → verify: nav에서 `/go` 진입.
4. 생성 모달(`MeetupCreateModal`) + 미니지도(**줌 컨트롤**) + `useLocationPicker` reverse geocoding → verify: 모달 내 핀 클릭 시 주소 자동표시, 저장 후 메인 지도에 주제색 핀 등장.

7. 자동만료(쿼리 `end_at>now()`) → verify: 종료시각 지난 모임 핀 없음.
8. 신고(`ReportModal`) + Join 알림(`common_notifications`) → verify: 신고 접수, 주최자 알림 수신.
9. 어드민 `/admin/go`(통계 카드·필터·테이블) + `(admin)/layout.tsx` 메뉴 + 상세 Drawer + 강제종료 2단계(`adminForceEndMeetup`) → verify: 행 클릭 Drawer, 강제종료 2단계 확인 후 모임 종료·참여자 알림.

## 9. 검증 (Verification)

**환경**: `.env.local` 에 `NEXT_PUBLIC_GOOGLE_MAPS_API_KEY`, `NEXT_PUBLIC_GOOGLE_MAPS_MAP_ID` 설정(Maps JS + Geocoding 활성화, 결제 등록).

**수동 시나리오** (`npm run dev` → `/go`):
1. 지도 nav 아래 전체 렌더, **줌인/줌아웃 버튼 동작**.
2. 번개모임 만들기 → 모달 미니지도에서 줌 후 핀 클릭 → 주소 자동표시 → 상세 한 줄 입력 → 저장.
3. 메인 지도에 주제 색 핀 등장(Realtime, 새로고침 없이).
4. 핀 클릭 → 우측 패널: 제목·주제·일시·장소·주최자/참여자 아바타·정원 프로그레스·참여하기.
5. 다른 계정 Join → 참여자 추가·주최자 알림·정원 도달 시 비활성·88%↑ 마감임박 뱃지.
6. 시작 전 취소 → 자리 반환. 종료시각 지난 모임 → 핀 사라짐.
7. 신고 모달 동작.
8. 어드민 `/admin/go` → 통계/필터/테이블, 행 클릭 Drawer, 강제종료 2단계 → 모임 종료·참여자 알림.

**빌드/타입**: `npm run gen:types` 후 `npm run build` 통과.
**테스트**: 프로젝트에 단위 테스트 관례가 약해 위 수동 시나리오를 1차 검증 기준으로 삼는다.
