# Kanto Trust Score (KTS) — 시스템 설계 문서

> Kanto 프로젝트 고유의 사용자 신뢰도 지표 시스템.
> 당근마켓 '당근온도'에서 영감을 받았으나, 온도 표현을 차용하지 않고 **점수(Score)** 형태로 차별화한다.

---

## 1. 기획 의도

현재 Kanto는 사용자 평판을 `avg_rating`(거래 후기 별점 평균) 하나로만 표현한다. 이것만으로는:

- 후기가 없는 신규 유저와 사기 전력이 있는 유저를 구분할 수 없다.
- 게시물 작성·구인공고·방렌트 등록 시 신뢰 있는 사용자를 우대할 수 없다.
- 필리핀 현지 기반 서비스 특성상 **거래 상대를 빠르게 신뢰 판단**해야 하는 니즈가 크다.

따라서 거래·활동·계정 신뢰도·위반 이력을 복합 반영한 **단일 숫자 지표**를 도입한다.

---

## 2. 시스템 개요

| 항목 | 내용 |
|------|------|
| 명칭 | **Kanto Trust Score** (줄여서 KTS) |
| 점수 범위 | 0 ~ 100점 |
| 시작 점수 | **36점** (신규 가입 시) |
| 표시 위치 | 내 프로필, 상대방 프로필 |
| 갱신 시점 | **배치 재계산** — Supabase pg_cron으로 하루 1~2회 전체 재계산 |
| 제재와의 관계 | **완전 독립 운영** — KTS가 높아도 욕설·사기 등 규정 위반 시 동일하게 제재 적용 |
| 공개 범위 | 본인 + 거래 상대방에게 공개 |

---

## 3. 점수 공식

```
KTS = clamp(36 + ReviewScore + TransactionScore + ActivityScore + TrendScore − PenaltyScore, 0, 100)
```

| 구성 요소 | 범위 |
|---------|------|
| ReviewScore | ±25점 |
| TransactionScore | 0 ~ +15점 |
| ActivityScore | 0 ~ +10점 |
| TrendScore | 0 ~ +14점 |
| PenaltyScore | 감점 (결과 0 클램프) |
| **이론상 최대** | **36 + 25 + 15 + 10 + 14 = 100점** |

모든 구성 요소는 **상한/하한 내에서 독립 계산** 후 합산한다.

---

### 3-1. ReviewScore — 거래 후기 점수 `±25점`

거래 후기(`reviews` 테이블, `deleted_at IS NULL`)의 평균 별점을 기반으로 산출.

```
RawScore    = (avg_rating − 3.0) / 2.0 × 25
ReviewScore = RawScore × confidence_weight
```

**평균 별점별 RawScore**

| avg_rating | RawScore |
|-----------|----------|
| 5.0 | +25점 |
| 4.0 | +12.5점 |
| 3.0 (중립) | 0점 |
| 2.0 | −12.5점 |
| 1.0 | −25점 |

**confidence_weight — 후기 수 신뢰 보정**

후기 1건으로 만점을 받아 즉시 최고 신뢰도가 되는 것을 방지한다.

| 후기 수 | 가중치 |
|--------|--------|
| 0건 | 0.0 (후기 없으면 ReviewScore = 0) |
| 1–3건 | 0.5 |
| 4–9건 | 0.75 |
| 10–19건 | 0.9 |
| 20건 이상 | 1.0 |

---

### 3-2. TransactionScore — 거래 완료 점수 `0 ~ +15점`

완료된 거래(`transactions.status = 'released'`) 수를 기반으로 산출.

```
TransactionScore = min(완료거래수 × 1.5, 15)
```

거래 취소율이 50% 이상이면 `TransactionScore × 0.5` 페널티 적용.

| 완료 거래 수 | 점수 |
|------------|------|
| 0건 | 0점 |
| 5건 | 7.5점 |
| 10건 이상 | 15점 (상한) |

---

### 3-3. ActivityScore — 활동 신뢰 점수 `0 ~ +10점`

#### 계정 나이 (최대 +6점)

```
AgeScore = min(가입 후 개월수 / 12 × 6, 6)
```

| 가입 기간 | 점수 |
|----------|------|
| 1개월 | +0.5점 |
| 6개월 | +3점 |
| 12개월 이상 | +6점 (상한) |

#### 게시물 활동 (최대 +4점)

```
PostScore = min(post_count × 0.4, 4)
```

| 게시물 수 | 점수 |
|---------|------|
| 5개 | +2점 |
| 10개 이상 | +4점 (상한) |

---

### 3-4. TrendScore — 주간 추이 보너스 `0 ~ +14점`

최근 4주간 주별 등급 스냅샷을 기반으로 활동 방향성을 평가한다.

```
TrendScore = GradeBase × TrendCoefficient
```

**GradeBase — 4주 내 최고 도달 등급 기준**

| 최고 등급 | GradeBase |
|---------|-----------|
| E | 0 (TrendScore 강제 0) |
| D | 4 |
| C | 8 |
| B 이상 | 14 |

**TrendCoefficient — 등급 변화 패턴 기준**

| 패턴 | 조건 | 계수 | TrendScore 예시 (최고=B) |
|------|------|------|--------------------------|
| 상승세 | 2주 이상 연속 등급 상승 | 1.0 | 14점 |
| 안정 유지 | 3주 이상 동일 등급 유지 | 1.0 | 14점 |
| 간헐적 | 등급이 오르내리며 방향 불명확 | 0.35 | ≈5점 |
| 하락세 | 2주 이상 연속 등급 하락 | 0.20 | ≈3점 |
| E등급 포함 | 현재 등급이 E | 0 | 0점 |

**산출 예시**

| 사용자 | 4주 패턴 | TrendScore |
|--------|---------|------------|
| 사용자 1 | D → C → B (상승세) | 14 × 1.0 = **14점** |
| 사용자 2 | B → B → B (안정 유지) | 14 × 1.0 = **14점** |
| 사용자 3 | B → C → B (간헐적) | 14 × 0.35 = **≈5점** |
| 사용자 4 | B → C → D (하락세) | 14 × 0.20 = **≈3점** |

> 신규 가입 후 4주 미만은 히스토리가 없으므로 TrendScore = 0.

**필요 데이터: 주간 스냅샷 테이블**

```sql
user_trust_history
  user_id     bigint
  week_date   date       -- 해당 주 시작일 (월요일)
  kts_score   integer
  grade_level integer    -- 0=E, 1=D, 2=C, 3=B, 4=A
```

배치 재계산(pg_cron) 실행 시 점수 저장과 TrendScore 계산을 함께 수행한다.

---

### 3-5. PenaltyScore — 위반 이력 감점 (최종 결과 0 클램프)

| 항목 | 감점 |
|------|------|
| 인정된 신고 (`status = 'resolved'`) 1건 | −8점 |
| 7일 정지 제재 이력 1회 | −5점 |
| 30일 정지 제재 이력 1회 | −15점 |

- `dismissed`(무혐의) 신고는 감점 없음
- `pending`(처리 중) 신고는 감점 없음 — 허위 신고 악용 방지
- 제재 이력은 **제재 해제 후에도 영구 반영** (이력 기반)

---

## 4. 등급 체계

| 점수 구간 | 등급 |
|---------|------|
| 90–100점 | **A** |
| 75–89점 | **B** |
| 50–74점 | **C** |
| 30–49점 | **D** |
| 0–29점 | **E** |

---

## 5. 예상 점수 분포

**유저 유형별 예상 KTS**

| 유형 | 예상 KTS | 등급 |
|------|---------|------|
| 신규 가입 (0–4주, 거래 없음) | ~37점 | D |
| D등급 4주 안정 | ~45점 | D |
| 3개월, C등급 안정 | ~59점 | C |
| 6개월, C→B 상승세 | ~72점 | C |
| 간헐적 활동 (B↔C 반복) | ~63점 | C |
| 하락세 유저 | ~54점 | C |
| 12개월, B등급 안정 유지 | ~87점 | B |
| 18개월+, 별점 4.5, 거래 20건+ | ~94점 | A |
| 신고 누적 + 제재 이력 | ~19점 | E |

**전체 분포 예상**

```
A  (90–100점) ████                   7%
B  (75–89점)  ██████████████        22%
C  (50–74점)  ███████████████████████████████  45%
D  (30–49점)  ██████████████        22%
E  (0–29점)   ████                   4%
```

C등급이 전체의 **45%** 를 차지하며 분포의 중심축을 이룬다.
A등급은 장기 활동 + 높은 거래 품질 + 일관된 상승세를 모두 충족한 유저에게만 부여된다.
