# 핫픽스 브랜치 규칙

## 브랜치 네이밍

```
hotfix-kanto-{이슈번호}
```

예: `hotfix-kanto-121`

## 기본 규칙

1. **범위 제한** — 해당 이슈의 버그 수정만 커밋한다. 기능 추가, 리팩토링 금지.
2. **베이스 브랜치** — `develop`에서 분기하고, 완료 후 `develop`으로 PR.
3. **커밋 메시지** — `fix: 수정 내용 한 줄 요약` 형식 사용.
4. **PR 전 확인** — 수정한 기능이 실제로 동작하는지 로컬에서 직접 확인 후 PR.
5. **리뷰어** — 최소 1인 승인 필요.

## 금지 사항

- 관련 없는 파일 수정
- 스타일/포맷 정리 포함
- 여러 이슈를 하나의 핫픽스 브랜치에서 처리
