## 1. 왜 접근성이 중요한가

접근성은 "장애인을 위한 배려"가 아니라 **모든 사용자를 위한 품질 기준**이다.

| 사용자 유형 | 영향 |
|------------|------|
| 스크린리더 사용자 (시각 장애) | aria-label, 시맨틱 태그 없으면 내용 파악 불가 |
| 키보드 전용 사용자 (운동 장애) | `div onClick`은 Tab 키로 접근 불가 |
| 저시력 사용자 | 색상 대비 기준 미충족 시 콘텐츠 인식 불가 |
| 모바일 보이스오버 사용자 | 버튼 역할/상태 미전달 시 조작 불가 |

**비즈니스 관점에서도:**
- 일부 국가에서 법적 의무 (ADA, EAA), 한국 「장애인차별금지법」 준수 대상
- SEO 향상 — 시맨틱 마크업은 검색엔진 크롤링에 직접 영향
- 코드 품질 향상 — 접근성을 고려한 마크업은 유지보수성도 높음

### 3-1. 시맨틱 HTML — 헤딩 계층 구조

**문제 파일:** `src/app/(user)/usedgoods/[id]/_components/UsedGoodsDetail.tsx`

#### 개선 전

```tsx
// 실제 제목이 p 태그 — 스크린리더가 "제목"으로 인식 못 함
<p className="text-3xl font-semibold">{data.posts.title}</p>

// 같은 페이지에 h1이 4개
<h1>상품 정보</h1>
<h1>판매자 정보</h1>
<h1>상품 설명</h1>
<h1>관련 매물</h1>
```

#### 개선 후

```tsx
<h1 className="text-3xl font-semibold">{data.posts.title}</h1>

<h2>상품 정보</h2>
<h2>판매자 정보</h2>
<h2>상품 설명</h2>
<h2>관련 매물</h2>
```

#### 올바른 헤딩 계층 원칙

```
h1 — 페이지당 정확히 1개. 페이지의 핵심 제목.
  h2 — 주요 섹션 구분
    h3 — h2의 하위 항목
```

#### 장단점 비교

| | 개선 전 | 개선 후 |
|--|--------|--------|
| 스크린리더 탐색 | "제목 없음"으로 읽힘 | h1→h2 구조로 즉시 페이지 파악 |
| SEO | 핵심 키워드 분산 | h1에 상품명 집중 |
| 개발 비용 | — | 태그 이름만 변경 |

> **핵심:** 태그는 **의미**를 표현하고, 시각적 크기는 **className**으로 조절한다.  
> `<h2 className="text-3xl font-bold">` — 크게 보이지만 h2로 계층 유지

### 3-2. 시맨틱 HTML — `<time>` 태그

**문제 파일:** `ContentCard.tsx`, `JobTitle.tsx`, `PostInfo.tsx`, `UsedGoodsDetail.tsx`, `NotificationItem.tsx`, `MessageList.tsx`

#### 개선 전

```tsx
// ContentCard.tsx / JobTitle.tsx / PostInfo.tsx / UsedGoodsDetail.tsx
<span>{formatTimeAgo(createdAt)}</span>

// NotificationItem.tsx — ISO 형식 날짜를 p 태그로 직접 렌더링
<p className="text-xs text-gray-400 mt-0.5">{n.created_at}</p>

// MessageList.tsx — 메시지 시간 + 날짜 구분선 모두 span
<span className="text-xs text-gray-400">
  {formatMessageTime(msg.created_at)}
</span>
<span className="text-xs text-gray-400">
  {formatDateDivider(msg.created_at)}
</span>
```

#### 개선 후

```tsx
// 상대 시간 표시 (모든 카드/상세 페이지 공통)
<time dateTime={createdAt}>
  {formatTimeAgo(createdAt)}
</time>

// 알림 생성일
<time dateTime={n.created_at} className="text-xs text-gray-400 mt-0.5">
  {formatTimeAgo(n.created_at)}
</time>

// 채팅 메시지 시간
<time dateTime={msg.created_at} className="text-xs text-gray-400">
  {formatMessageTime(msg.created_at)}
</time>

// 날짜 구분선
<time dateTime={msg.created_at} className="text-xs text-gray-400">
  {formatDateDivider(msg.created_at)}
</time>
```

#### 파일별 누락 위치 전체 목록

| 파일 | 사용 함수/데이터 | 위치 |
|------|----------------|------|
| `ContentCard.tsx` | `formatTimeAgo(createdAt)` | 카드 하단 시간 표시 |
| `JobTitle.tsx` | `formatTimeAgo(job.posts.created_at)` | Clock 아이콘 옆 |
| `PostInfo.tsx` (rental) | `formatTimeAgo(rental.created_at)` | Clock 아이콘 옆 |
| `UsedGoodsDetail.tsx` | `formatTimeAgo(data.posts.created_at)` | 상세 페이지 시간 |
| `NotificationItem.tsx` | `n.created_at` 직접 출력 | 알림 생성 시각 |
| `MessageList.tsx` | `formatMessageTime(msg.created_at)` | 메시지 전송 시각 |
| `MessageList.tsx` | `formatDateDivider(msg.created_at)` | 날짜 구분선 |

#### 장단점 비교

| | 개선 전 | 개선 후 |
|--|--------|--------|
| 스크린리더 | "3시간 전" 텍스트만 읽음 | 정확한 날짜+시간 맥락 제공 |
| 검색엔진 | 날짜 데이터로 인식 못 함 | 발행일 구조화 데이터 활용 가능 |
| 개발 비용 | — | `<span>` → `<time dateTime={...}>` 교체만으로 완료 |

### 3-4. 아이콘 전용 버튼 — `aria-label`

#### 개선 전

```tsx
// 스크린리더: "버튼" 이라고만 읽힘 — 무슨 버튼인지 알 수 없음
<button><Heart className="w-4 h-4" /></button>
<button><Share2 className="w-4 h-4" /></button>
```

#### 개선 후

```tsx
<button aria-label="좋아요"><Heart className="w-4 h-4" /></button>
<button aria-label="공유하기"><Share2 className="w-4 h-4" /></button>

// 토글 버튼: 상태에 따라 동적으로 변경
<button
  aria-label={isLiked ? "좋아요 취소" : "좋아요"}
  aria-pressed={isLiked}
  onClick={handleLike}
>
  <Heart className={isLiked ? "fill-red-400" : ""} />
</button>

// 비밀번호 표시 토글
<button
  type="button"
  aria-label={showPassword ? "비밀번호 숨기기" : "비밀번호 표시"}
  aria-pressed={showPassword}
  onClick={() => setShowPassword(v => !v)}
>
  <EyeIcon visible={showPassword} />
</button>
```

### 3-5. 메뉴/드롭다운 토글 — `aria-expanded` / `aria-haspopup`

#### 개선 전

```tsx
// Header.tsx — 프로필 드롭다운 (isProfileOpen 상태 있지만 aria 없음)
<Button onClick={() => setIsProfileOpen(v => !v)}>
  <User />
</Button>

// Header.tsx — 모바일 햄버거 (isMobileOpen 상태 있지만 aria 없음)
<Button onClick={() => setIsMobileOpen(v => !v)}>
  {isMobileOpen ? <X /> : <Menu />}
</Button>

// NotificationBell.tsx — 알림 드롭다운
<Button onClick={handleToggle}>
  <Bell />
</Button>

// MainSearchBar.tsx — 카테고리 드롭다운
<button onClick={() => setDropdownOpen(v => !v)}>
  {selectedCategory.label}<ChevronDown />
</button>

// ChatHeader.tsx — 더보기 메뉴
<button onClick={() => setMenuOpen(prev => !prev)}>
  <MoreVertical />
</button>

// FilterDropdown.tsx — 필터 드롭다운
<button onClick={() => setOpen(v => !v)}>
  {label}<ChevronDown />
</button>

// SearchBar.tsx — 지역 드롭다운
<button onClick={() => setLocationDropdownOpen(v => !v)}>
  {selectedLocationLabel}<ChevronDown />
</button>
```

#### 개선 후

```tsx
// Header.tsx — 프로필 드롭다운
<Button
  onClick={() => setIsProfileOpen(v => !v)}
  aria-label="내 프로필 메뉴"
  aria-expanded={isProfileOpen}
  aria-haspopup="menu"
>
  <User />
</Button>

// Header.tsx — 모바일 햄버거
<Button
  onClick={() => setIsMobileOpen(v => !v)}
  aria-label={isMobileOpen ? "메뉴 닫기" : "메뉴 열기"}
  aria-expanded={isMobileOpen}
  aria-controls="mobile-nav"
>
  {isMobileOpen ? <X /> : <Menu />}
</Button>
<nav id="mobile-nav" {/* aria-controls 대상에 id 추가 */}>

// NotificationBell.tsx
<Button
  onClick={handleToggle}
  aria-label="알림"
  aria-expanded={isOpen}
  aria-haspopup="true"
>
  <Bell />
  {Badge}
</Button>

// MainSearchBar.tsx — 데스크탑 카테고리 드롭다운
<button
  onClick={() => setDropdownOpen(v => !v)}
  aria-expanded={dropdownOpen}
  aria-haspopup="listbox"
>
  {selectedCategory.label}<ChevronDown />
</button>

// MainSearchBar.tsx — 모바일 바텀시트 트리거
<button
  onClick={() => setBottomSheetOpen(true)}
  aria-expanded={bottomSheetOpen}
  aria-haspopup="dialog"
>
  {selectedCategory.label}<ChevronDown />
</button>

// ChatHeader.tsx
<button
  onClick={() => setMenuOpen(prev => !prev)}
  aria-label="더보기 메뉴"
  aria-expanded={menuOpen}
  aria-haspopup="menu"
>
  <MoreVertical />
</button>

// FilterDropdown.tsx
<button
  onClick={() => setOpen(v => !v)}
  aria-expanded={open}
  aria-haspopup="listbox"
>
  {label}<ChevronDown />
</button>

// SearchBar.tsx — 데스크탑 지역 드롭다운
<button
  onClick={() => setLocationDropdownOpen(v => !v)}
  aria-expanded={locationDropdownOpen}
  aria-haspopup="listbox"
>
  {selectedLocationLabel}<ChevronDown />
</button>

// SearchBar.tsx — 모바일 지역 바텀시트
<button
  onClick={() => setMobileLocationOpen(true)}
  aria-expanded={mobileLocationOpen}
  aria-haspopup="dialog"
>
  {selectedLocationLabel}<ChevronDown />
</button>
```

| `MainSearchBar.tsx` | 카테고리 바텀시트 (모바일) | `aria-expanded={bottomSheetOpen}` + `aria-haspopup="dialog"` |
| `ChatHeader.tsx` | MoreVertical 메뉴 버튼 | `aria-expanded={menuOpen}` + `aria-haspopup="menu"` |
| `FilterDropdown.tsx` | 필터 드롭다운 버튼 | `aria-expanded={open}` + `aria-haspopup="listbox"` |
| `SearchBar.tsx` | 지역 드롭다운 (데스크탑) | `aria-expanded={locationDropdownOpen}` + `aria-haspopup="listbox"` |
| `SearchBar.tsx` | 지역 바텀시트 (모바일) | `aria-expanded={mobileLocationOpen}` + `aria-haspopup="dialog"` |

#### 장단점 비교

| | 개선 전 | 개선 후 |
|--|--------|--------|
| 스크린리더 | 클릭 후에야 열림 여부 파악 | "필터, 메뉴 있음, 접혀 있음" 사전 안내 |
| 상태 변경 | 사용자가 알 수 없음 | "펼쳐져 있음" 자동 고지 |

### 3-6. 클릭 가능 요소 — `<div onClick>` → `<Link>` / `<button>`

**문제 파일:** `JobCard.tsx`, `UsedGoodsDetail.tsx` (관련 매물), `signup/page.tsx`

#### 개선 전

```tsx
// JobCard.tsx — 카드 전체 div에 onClick
<div onClick={() => router.push(`/job/${id}`)} className="cursor-pointer">
  {/* 카드 내용 */}
</div>

// UsedGoodsDetail.tsx — 관련 매물 이미지
<div onClick={() => router.push(`/usedgoods/${item.id}`)} className="cursor-pointer">
  <Image ... />
</div>

// signup/page.tsx — div로 커스텀 체크박스 구현
<div onClick={handleToggleAll} className="cursor-pointer">
  <div className="w-4 h-4 rounded border-2 ...">
    {allChecked && <svg>...</svg>}
  </div>
  <span>전체 동의하기</span>
</div>

{AGREES.map((a) => (
  <div key={a.id} onClick={() => handleItemClick(a.id)} className="cursor-pointer">
    {/* ... */}
  </div>
))}
```

#### 개선 후

```tsx
// JobCard.tsx
<Link href={`/job/${id}`}>
  {/* 카드 내용 */}
</Link>

// UsedGoodsDetail.tsx
<Link href={`/usedgoods/${item.id}`}>
  <Image alt={`${item.title} 상품 이미지`} ... />
</Link>

// signup/page.tsx — 네이티브 input[type="checkbox"] + label 로 교체
<label className="flex items-center gap-3 bg-gray-50 rounded-lg px-4 py-3 cursor-pointer">
  <input
    type="checkbox"
    checked={allChecked}
    onChange={handleToggleAll}
    className="w-4 h-4 accent-teal-500"
  />
  <span className="text-sm font-medium text-gray-900">전체 동의하기</span>
</label>

{AGREES.map((a) => (
  <label key={a.id} className="flex items-start gap-3 px-4 py-2 cursor-pointer hover:bg-gray-50 rounded-lg">
    <input
      type="checkbox"
      checked={agreed[a.id as keyof typeof agreed]}
      onChange={() => handleItemClick(a.id)}
      className="mt-0.5 w-4 h-4 accent-teal-500"
    />
    <div className="flex-1">
      <div className="flex items-center gap-2">
        <span className="text-sm text-gray-900">{a.label}</span>
        <span className={`text-xs px-2 py-0.5 rounded-full font-medium ${a.required ? "bg-teal-100 text-teal-600" : "bg-gray-100 text-gray-500"}`}>
          {a.required ? "필수" : "선택"}
        </span>
      </div>
      <p className="text-xs text-gray-500 mt-0.5">{a.desc}</p>
    </div>
  </label>
))}
```

#### 판단 기준

```
클릭 후 페이지 이동 → <Link href="...">
클릭 후 상태 변경 → <button type="button">
토글 ON/OFF → <input type="checkbox"> + <label>
절대 사용 금지 → <div onClick>, <span onClick>, <p onClick>, <img onClick>
```

#### 장단점 비교

| | 개선 전 | 개선 후 |
|--|--------|--------|
| 키보드 접근 | Tab 불가, Enter 불가 | Tab → Enter 동작 가능 |
| 스크린리더 | 역할 미전달 | "링크", "버튼", "체크박스" 정확히 전달 |
| 체크박스 상태 | aria-checked 없음 | `<input type="checkbox">`가 자동 처리 |
| 우클릭 새 탭 | 불가 | Link 사용 시 Ctrl+클릭 가능 |

### 3-7. 이미지 클릭 — `<img onClick>` → `<button>`

**문제 파일:** `UsedGoodsDetail.tsx`, `ImageCarresel.tsx` (썸네일)

#### 개선 전

```tsx
<Image
  src={image}
  alt={`썸네일 ${index + 1}`}
  width={64}
  height={64}
  onClick={() => setCurrentIndex(index)}
  className="cursor-pointer"
/>
```

#### 개선 후

```tsx
<button
  type="button"
  onClick={() => setCurrentIndex(index)}
  aria-label={`${index + 1}번째 이미지로 이동`}
  aria-pressed={currentIndex === index}

  />

```

#### 장단점 비교

| | 개선 전 | 개선 후 |
|--|--------|--------|
| 키보드 접근 | 불가 | Tab + Enter로 썸네일 선택 가능 |
| 현재 선택 상태 | 시각적으로만 표시 | `aria-pressed`로 스크린리더에 전달 |
| 포커스 링 | 없음 | `focus-visible:ring`으로 표시 |

### 3-8. 이미지 alt 텍스트 품질

**문제 파일:** `UsedGoodsDetail.tsx`, `ImageCarresel.tsx`

#### 개선 전

```tsx
<Image src={images[currentIndex]} alt="대표 이미지" />
<Image src={images[currentIndex]} alt="숙소 이미지" />
```

#### 개선 후

```tsx
<Image

  alt={`${data.posts.title} — ${currentIndex + 1}번째 이미지`}
/>
<Image
  src={images[currentIndex]}
```

#### alt 텍스트 작성 원칙

| 상황 | alt 내용 |
|------|---------|
| 상품/게시글 이미지 | `{제목} — {N}번째 이미지` |

### 3-9. 탭 UI — `role="tablist"` / `role="tab"` / `aria-selected`

**문제 파일:** `src/app/(user)/main/_components/popular/PopularTabs.tsx`

#### 개선 전

```tsx
// 탭처럼 보이지만 ARIA 탭 역할 없음 — 스크린리더가 탭인지 알 수 없음
<div className="flex gap-6 items-end">
  {TABS.map((tab) => (
    <button
      key={tab.key}
      onClick={() => setActiveTab(tab.key)}
      className={activeTab === tab.key ? "text-black underline" : "text-gray-400"}
    >
      {tab.label}
    </button>
  ))}
</div>

<div className="grid grid-cols-4 gap-4">
  {activeItems.map((item) => <MainCard key={item.id} item={item} />)}
</div>
```

#### 개선 후

```tsx
// role="tablist" — 탭 목록임을 명시
<div role="tablist" aria-label="인기 카테고리" className="flex gap-6 items-end">
  {TABS.map((tab) => (
    <button
      key={tab.key}
      role="tab"
      aria-selected={activeTab === tab.key}
      aria-controls={`tab-panel-${tab.key}`}
      id={`tab-${tab.key}`}
      onClick={() => setActiveTab(tab.key)}
      className={activeTab === tab.key ? "text-black underline" : "text-gray-400"}
    >
      {tab.label}
    </button>
  ))}
</div>

// role="tabpanel" — 탭 패널임을 명시, aria-labelledby로 탭과 연결
<div
  role="tabpanel"
  id={`tab-panel-${activeTab}`}
  aria-labelledby={`tab-${activeTab}`}
  className="grid grid-cols-4 gap-4"
>
  {activeItems.map((item) => <MainCard key={item.id} item={item} />)}
</div>
```

#### 장단점 비교

| | 개선 전 | 개선 후 |
|--|--------|--------|
| 스크린리더 | "버튼 3개" (탭인지 알 수 없음) | "중고거래 탭, 1/3, 선택됨" |
| 키보드 탐색 | Tab 키로만 이동 | Arrow 키로 탭 간 이동 (APG 패턴) |
| WCAG 준수 | ❌ | ✅ ARIA Authoring Practices 준수 |

### 3-10. 폼 접근성 — 입력 라벨 및 커스텀 체크박스

**문제 파일:** `ChatInput.tsx`, `MainSearchBar.tsx`, `SearchBar.tsx`

#### 개선 전

```tsx
// ChatInput.tsx — 텍스트 입력 필드에 라벨 없음
<input
  type="text"
  value={input}
  placeholder="메시지를 입력하세요..."
  className="flex-1 text-sm bg-gray-50 rounded-full px-4 py-2 outline-none"
/>

// MainSearchBar.tsx / SearchBar.tsx — 검색 입력에 라벨 없음
<input
  type="text"
  placeholder="검색어를 입력해주세요"
  className="min-w-0 flex-1 ..."
/>
```

#### 개선 후

```tsx
// 방법 1: aria-label 직접 추가 (시각적 라벨이 불필요할 때)
<input
  type="text"
  aria-label="메시지 입력"
  value={input}
  placeholder="메시지를 입력하세요..."
  className="flex-1 text-sm ..."
/>

// 방법 2: sr-only label (스크린리더에만 전달)
<label htmlFor="search-input" className="sr-only">검색어 입력</label>
<input
  id="search-input"
  type="text"
  placeholder="검색어를 입력해주세요"
  className="min-w-0 flex-1 ..."
/>
```

#### 파일별 누락 위치

| 파일 | 입력 필드 | 추가할 라벨 |
|------|----------|-----------|
| `ChatInput.tsx` | 메시지 텍스트 input | `aria-label="메시지 입력"` |
| `MainSearchBar.tsx` | 검색어 input | `aria-label="검색어 입력"` |
| `SearchBar.tsx` | 검색어 input ×2 (모바일/데스크탑) | `aria-label="검색어 입력"` |

> `placeholder`는 라벨의 대체제가 아니다. placeholder는 포커스 시 사라지고, 고대비 모드에서 보이지 않을 수 있으며, 모든 스크린리더가 항상 읽어주지는 않는다.

#### 장단점 비교

| | 개선 전 | 개선 후 |
|--|--------|--------|
| 스크린리더 | "텍스트 입력" (무맥락) | "메시지 입력, 텍스트 필드" |
| 고대비 모드 | placeholder 안 보임 | aria-label은 항상 유지 |
| 포커스 후 힌트 | placeholder 사라짐 | aria-label 유지 + placeholder 힌트 병행 가능 |

## 4. 팀 코딩 규칙

> 이 규칙은 PR 리뷰 기준이 됩니다. 신규 컴포넌트 작성 시 반드시 준수해야 합니다.

---

### Rule 1 — 헤딩 계층 구조

```
✅ 각 페이지(route)에 <h1>은 정확히 1개
✅ h1 → h2 → h3 순서를 건너뛰지 않을 것
✅ 시각적 크기는 className으로, 계층은 태그로 표현할 것

❌ 시각적으로 크게 만들려고 h1 남용
❌ 시각적으로 작게 만들려고 h2를 p 태그로 대체
```

---

### Rule 2 — 시맨틱 태그 사용

```
✅ 날짜/시간 → <time dateTime="ISO8601형식">
✅ key-value 목록 → <dl><dt><dd>
✅ 내비게이션 → <nav aria-label="설명">
✅ 주요 콘텐츠 → <main>
✅ 독립 콘텐츠 블록 → <article> 또는 <section aria-labelledby>

❌ 모든 것을 <div>, <span>, <p>로 처리
```

---

### Rule 3 — 인터랙티브 요소

```
✅ 페이지 이동 → <Link href="...">
✅ 동작 실행 → <button type="button">
✅ 폼 제출 → <button type="submit">
✅ 체크박스 → <input type="checkbox"> + <label> (div 커스텀 체크박스 금지)
✅ form 내부 버튼은 반드시 type 명시

❌ <div onClick>, <span onClick>, <p onClick>, <img onClick>
❌ type 없는 form 내부 <button> (기본값이 submit이어서 의도치 않게 제출됨)
```

---

### Rule 4 — aria-label 규칙

```
✅ 아이콘 전용 버튼 → 반드시 aria-label 추가
✅ 토글 버튼 (좋아요/비밀번호 표시 등) → aria-pressed={boolean}
✅ 드롭다운 토글 → aria-expanded={boolean} + aria-haspopup
✅ 상태에 따라 동적으로 변하는 레이블 작성

판단 기준: 눈 감고 들었을 때 무슨 동작인지 알 수 있는가?
→ 알 수 없으면 aria-label 필수

❌ 텍스트가 있는 버튼에 aria-label 중복 (텍스트가 이미 이름 역할)
```

---

### Rule 5 — 탭 UI

```
✅ 탭 컨테이너 → role="tablist" + aria-label
✅ 개별 탭 버튼 → role="tab" + aria-selected + aria-controls
✅ 탭 패널 → role="tabpanel" + aria-labelledby
✅ id로 tab ↔ panel 연결

❌ 시각적으로만 탭처럼 보이는 button 목록 (ARIA 역할 없음)
```

---

### Rule 6 — 입력 필드 라벨

```
✅ 모든 <input>/<textarea>/<select>는 연결된 <label> 또는 aria-label 필수
✅ 시각적으로 라벨이 없어도 → aria-label 또는 sr-only label
✅ placeholder는 힌트용으로만 사용 (라벨 대체 불가)

❌ placeholder만 있고 label/aria-label 없는 입력 필드
```

---

### Rule 7 — 이미지 alt 텍스트

```
✅ 콘텐츠 이미지 → 구체적 설명 (제목, 순서 포함)
✅ 장식 이미지 → alt="" + role="presentation"
✅ 버튼 내부 아이콘 이미지 → alt="" (버튼의 aria-label이 설명)

❌ alt="이미지" / alt="사진" / alt="대표 이미지" (무의미한 설명)
❌ alt 속성 누락 (HTML 유효성 오류)
```

---

### Rule 8 — 포커스 관리

```
✅ 모달 열릴 때 → 모달 내부 첫 번째 포커스 가능 요소로 이동
✅ 모달 닫힐 때 → 모달을 열었던 버튼으로 포커스 복귀
✅ ESC 키로 모달/드롭다운 닫기
✅ focus-visible:ring 사용 (키보드 사용자에게만 포커스 링 표시)

❌ outline: none; 단독 사용 (포커스 링 완전 제거)
→ 대신: focus:outline-none focus-visible:ring-2 focus-visible:ring-teal-400
```

---

### Rule 9 — 색상 대비

```
✅ 일반 텍스트: 명암비 4.5:1 이상 (WCAG AA)
✅ 큰 텍스트(18pt+): 명암비 3:1 이상

주의:
- text-gray-400 on white → 약 2.9:1 ❌ (본문 텍스트 WCAG 미충족)
- text-gray-500 on white → 약 4.6:1 ✅
- PopularTabs 비활성 탭 text-gray-400 → gray-500으로 교체 권장
```

## 5. PR 리뷰 체크리스트

새 컴포넌트 또는 페이지 PR 시 아래 항목을 셀프 체크 후 제출.

### 마크업 구조

- [ ] 페이지에 `<h1>`이 정확히 1개인가?
- [ ] 헤딩 계층(h1→h2→h3)이 올바른가?
- [ ] 날짜/시간에 `<time dateTime="">` 사용했는가?
- [ ] key-value 데이터에 `<dl><dt><dd>` 사용했는가?

### 인터랙티브 요소

- [ ] 페이지 이동은 `<Link>`를 사용했는가? (`div onClick` 없는가?)
- [ ] form 내부 버튼에 `type` 속성이 있는가?
- [ ] 체크박스가 `<input type="checkbox">` + `<label>`로 구현되었는가?
- [ ] `<img onClick>` 없는가? (button으로 감쌌는가?)

### aria

- [ ] 아이콘 전용 버튼에 `aria-label`이 있는가?
- [ ] 토글 버튼에 `aria-pressed`가 있는가?
- [ ] 드롭다운 토글에 `aria-expanded` + `aria-haspopup`이 있는가?
- [ ] 탭 UI에 `role="tablist"` / `role="tab"` / `aria-selected`가 있는가?
- [ ] Tab 키로 모든 기능에 접근할 수 있는가?

### 폼

- [ ] 모든 입력 필드에 `aria-label` 또는 연결된 `<label>`이 있는가?
- [ ] placeholder만 있고 label이 없는 input이 없는가?

### 이미지

- [ ] 모든 `<Image>`에 `alt` 속성이 있는가?
- [ ] alt 텍스트가 "이미지", "사진" 같은 무의미한 단어가 아닌가?
- [ ] 장식 이미지에 `alt=""` + `role="presentation"`이 있는가?
