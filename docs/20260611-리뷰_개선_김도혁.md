## 4. [1차 필수] useChatMessages 읽음처리 useEffect 의존성 추가 ✅

- [src/hooks/chat/useChatMessages.ts](../src/hooks/chat/useChatMessages.ts) `:38`

```ts
// before
}, []);  // 또는 의존성 누락

// after

```

## 5. [3차 필수] Notion API 실패 시 fallback 추가 ✅

`getNotionContent`에서 Notion API 호출(`pageToMarkdown`) 실패 시 페이지 전체가 500으로 죽는 문제 수정.
`policy`, `service`, `privacy`, `youth` 4개 약관 페이지가 동시에 보호됨.

**수정 파일**
- [src/services/notion/notion.ts](../src/services/notion/notion.ts)

```ts
// before — API 실패 시 throw → 페이지 500 에러
export async function getNotionContent(pageId: string) {
  const mdBlocks = await n2m.pageToMarkdown(pageId);
  const mdString = n2m.toMarkdownString(mdBlocks);
  return mdString.parent;
}

// after — 실패 시 fallback 문자열 반환
export async function getNotionContent(pageId: string) {
  try {
    const mdBlocks = await n2m.pageToMarkdown(pageId);
    const mdString = n2m.toMarkdownString(mdBlocks);
    return mdString.parent;
  } catch {
    return "현재 내용을 불러올 수 없습니다. 잠시 후 다시 시도해주세요.";
  }
}
```

## 8. [1차 제안] useSpamPrevention setInterval cleanup 추가 ✅

쿨다운 도중 컴포넌트 언마운트(채팅방 이탈) 시 `setInterval`이 계속 실행되는 메모리 누수 수정.

- `intervalRef`로 interval ID를 보관
- `useEffect` cleanup에서 언마운트 시 `clearInterval` 호출
- 정상 종료(10초 경과) 시에는 기존처럼 `setCooldownSeconds` 내부에서 정리

**수정 파일**
- [src/hooks/chat/useSpamPrevention.ts](../src/hooks/chat/useSpamPrevention.ts)

```ts
// before — interval이 지역 변수라 언마운트 시 정리 불가
const interval = setInterval(() => { ... }, 1000);

// after — ref로 보관 + useEffect cleanup
const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null);
useEffect(() => {
  return () => { if (intervalRef.current) clearInterval(intervalRef.current); };
}, []);
intervalRef.current = setInterval(() => { ... }, 1000);
```
