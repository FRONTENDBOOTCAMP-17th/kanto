## 1. `users` 테이블 RLS — 개인정보 노출 차단 (가장 먼저)

지금은 publishable 키만으로 전체 회원 이메일·전화가 읽힙니다. Supabase SQL Editor에서 정책을 추가하세요.

```sql
-- 1) RLS 켜기 (이미 켜져 있으면 생략)
alter table public.users enable row level security;

-- 2) 본인 행만 select 허용 (auth_id = 로그인 사용자)
create policy "users_select_self"
on public.users for select
to authenticated
using (auth_id = auth.uid());
```

다른 사람의 **이름·아바타만** 공개로 보여줘야 한다면(예: 판매자 정보), 전체 테이블을 열지 말고 공개 컬럼만 담은 **뷰**를 만들어 그 뷰에만 select를 허용하세요.

```sql
create view public.public_profiles as
  select id, name, avatar_url, created_at from public.users;
-- 뷰에 대해서만 anon select 허용, users 원본은 본인만
```

적용 후 확인(이메일·전화가 더는 안 나와야 정상):
```bash
curl "$URL/rest/v1/users?select=email,phone" -H "apikey: $ANON_KEY"
# → []  또는 본인 행만
```

## 2. `usedgoods/[id]` 서버 조회로 교체 → 빌드 통과

지금 `usedgoods/[id]/page.tsx`는 서버 컴포넌트인데 **브라우저용 클라이언트**(`@/lib/supabase`)를 import 해서 쓰고, 타입도 안 맞아 `npm run build`가 실패합니다. rental에서 이미 만든 패턴처럼 **서비스 함수 + 서버 클라이언트**로 바꾸면 됩니다.

```ts
// src/services/usedGoods/usedGoods.ts (rental.ts와 같은 모양으로)
import { createClient } from "@/utils/supabase/server"; // 서버용 클라이언트

export async function getUsedGoodsDetail(id: number) {
  const supabase = await createClient();
  const { data, error } = await supabase
    .from("used_goods")
    .select(`*, posts (*, users (*))`)
    .eq("post_id", id)
    .single();
  if (error) return null;
  return data;
}
```

```tsx
// usedgoods/[id]/page.tsx
import { getUsedGoodsDetail } from "@/services/usedGoods/usedGoods";

export default async function UsedGoodsDetailPage({ params }: { params: Promise<{ id: string }> }) {
  const { id } = await params;
  const data = await getUsedGoodsDetail(Number(id));
  if (!data) return <div>상품을 찾을 수 없습니다.</div>;
  return <UsedGoodsDetail data={data} />;
}
```

타입 불일치(`posts.status: string`)는, 서비스 함수의 반환 타입을 **쿼리 결과 모양에 맞게 하나로 정의**해서 `UsedGoodsDetail`이 그 타입을 받게 하면 사라집니다. (rental에서 `rentalDetail.ts`로 정리한 그 방식을 그대로 적용)

마지막에 꼭 확인:
```bash
npx tsc --noEmit   # 에러 0
npm run build      # 통과
```

## 4. `useImageUpload` cleanup 누수 + 약관 Notion fallback

### 4-1. blob URL 누수 (stale closure)

지금 cleanup은 `[]` 의존성이라 **첫 렌더의 빈 배열**만 기억해서, 실제로 올린 미리보기 URL을 언마운트 때 못 지웁니다. ref로 항상 최신 배열을 가리키게 고치세요.

```ts
export function useImageUpload(maxCount = 10) {
  const [imageFiles, setImageFiles] = useState<File[]>([]);
  const [imagePreviews, setImagePreviews] = useState<string[]>([]);
  const previewsRef = useRef<string[]>([]);          // 추가
  previewsRef.current = imagePreviews;               // 렌더마다 최신화

  useEffect(() => {
    return () => {
      previewsRef.current.forEach((url) => URL.revokeObjectURL(url)); // 최신값으로 정리
    };
  }, []);
  // ... 나머지 동일
}
```
이러면 언마운트 시 그때까지 만든 URL을 전부 해제합니다. (`removeImage`에서 개별 revoke 하는 건 그대로 두면 됩니다.)

### 4-2. 약관 페이지가 Notion 실패에 통째로 죽지 않게

`getNotionContent`를 try/catch로 감싸 실패 시 fallback을 반환하세요.

```ts
export async function getNotionContent(pageId: string) {
  try {
    if (!pageId) return "약관 내용을 불러올 수 없습니다.";
    const mdBlocks = await n2m.pageToMarkdown(pageId);
    return n2m.toMarkdownString(mdBlocks).parent;
  } catch (e) {
    console.error("Notion fetch 실패:", e);
    return "약관 내용을 일시적으로 불러올 수 없습니다. 잠시 후 다시 시도해주세요.";
  }
}
```

또는 라우트 단위 에러 바운더리를 두는 방법:
```tsx
// src/app/(user)/terms/error.tsx
"use client";
export default function TermsError() {
  return <div className="p-8 text-center text-gray-400">약관을 불러오지 못했습니다. 잠시 후 다시 시도해주세요.</div>;
}
```
약관·정책은 "항상 보여야 하는" 페이지라, 외부 API 하나 때문에 5xx 화면이 뜨면 안 됩니다.

```ts
const { error } = target.isLiked
  ? await supabase.from("common_likes").delete()
      .eq("user_id", currentUserId).eq("target_type", "post").eq("target_id", id)
  : await supabase.from("common_likes")
      .insert({ user_id: currentUserId, target_type: "post", target_id: id });

if (error) {

  setItems((prev) => prev.map((item) =>
```
