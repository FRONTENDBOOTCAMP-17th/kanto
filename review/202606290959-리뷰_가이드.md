## 1. [필수] 메인 페이지 — 서버 컴포넌트에서 getTranslations로 전환

`src/app/(user)/main/page.tsx`는 `'use client'`가 없으니 서버 컴포넌트입니다. 서버 컴포넌트에서는 클라이언트 훅인 `useTranslations`를 못 씁니다. `next-intl/server`의 `getTranslations`를 `await`로 받아야 합니다.

지금 코드:

```tsx
import type { Metadata } from "next";
import { useTranslations } from "next-intl";          // ← 클라이언트 훅
// ...
export default function MainPage() {
  const t = useTranslations("Main");                  // ← 서버 컴포넌트에서 호출
  return (
    // ...
    <p>{t("tagline")}</p>
  );
}
```

이렇게 바꿉니다:

```tsx
import type { Metadata } from "next";
import { getTranslations } from "next-intl/server";   // ← 서버용
// ...
export default async function MainPage() {            // ← async 추가
  const t = await getTranslations("Main");            // ← await 추가
  return (
    // ...
    <p>{t("tagline")}</p>
  );
}
```

포인트는 세 가지입니다. import를 `next-intl/server`의 `getTranslations`로, 컴포넌트 함수를 `async`로, 호출을 `await getTranslations(...)`로. `t(...)`를 쓰는 JSX는 그대로 둬도 됩니다. `rental/page.tsx`·`usedgoods/page.tsx`가 이미 이 패턴을 쓰고 있으니 그 두 파일을 참고하면 됩니다.

### 겸사겸사 — 목록 중복 import 정리 [사소]

`src/app/(user)/rental/page.tsx:4-5`:

```tsx
import { getRentalList, getRentalBarangays } from "@/services/rental/rental";
import { getRentalList } from "@/services/rental/rental";   // ← 이 줄 삭제
```

## 2. [필수] 상세 페이지 JSON-LD — 닫히지 않은 div 제거

JSON-LD `<script>`를 끼워 넣으면서 여분의 래퍼 `<div>`가 한 겹 더 들어가 닫히지 않은 상태입니다. `<script>`는 이미 있는 최상위 컨테이너 안에 그대로 두고, 그 뒤로 원래 본문 구조가 바로 이어지게 하면 됩니다.

### job/[id]/page.tsx

지금(`:81~`):

```tsx
return (
  <div className="page-container w-full py-6">       {/* 새로 끼어든 래퍼 */}
    <script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }} />
    <div className="border ... divide-y ...">         {/* :86 */}
  <div className="page-container pb-12">              {/* :87 ← 원래 본문 컨테이너, 들여쓰기도 어긋남 */}
```

원래 본문은 `<div className="page-container pb-12">`로 시작했을 겁니다. 새 래퍼를 없애고 그 컨테이너 안에 `<script>`를 넣습니다:

```tsx
return (
  <div className="page-container pb-12">             {/* 원래 본문 컨테이너 하나만 */}
    <script type="application/ld+json" dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }} />
    {/* 이하 원래 본문(BackButton/VerifyAuthor 등)이 그대로 이어짐 */}
```

`:86`의 `<div ... divide-y ...>`가 원래 본문 어디에 속하는 줄인지 확인해서, 본문 흐름에 맞게 한 번만 열리도록 정리하면 됩니다(현재는 본문 컨테이너 앞에 중복으로 열려 있습니다).

### rental/[id]/page.tsx

지금(`:71~`):

```tsx
return (
  <div className="page-container pb-12">
    <script type="application/ld+json" ... />
    <div className="flex items-center justify-between">      {/* :76 ← 빈 여분 div */}
    <div className="flex items-center justify-between mt-4">  {/* :77 ← 실제 본문 시작 */}
      <BackButton />
```

`:76`의 `<div className="flex items-center justify-between">`가 닫히지도, 내용을 감싸지도 않는 여분입니다. 이 줄을 삭제하고 본문이 바로 `:77`부터 이어지게 합니다:

```tsx
return (
  <div className="page-container pb-12">
    <script type="application/ld+json" ... />
    <div className="flex items-center justify-between mt-4">
      <BackButton />
      {/* ... */}
```

### 확인 방법

수정 후 `npx tsc --noEmit`과 `npm run build`를 돌려 JSX 구조 에러가 사라졌는지 보면 확실합니다. div 짝이 안 맞으면 빌드가 잡아줍니다.

## 3. [필수] responsive-select 숨김 select — readOnly 또는 비제어로

`src/components/.../responsive-select.tsx:83-90`의 숨김 `<select required>`는 브라우저 기본 검증을 살리려고 둔 건데, `value`를 주면서 `onChange`를 빈 함수로 둬서 React가 controlled input 경고를 냅니다.

지금:

```tsx
<select
  aria-hidden
  tabIndex={-1}
  required
  disabled={disabled}
  value={value}
  onChange={() => {}}     // ← 빈 핸들러 = 경고 원인
  className="..."
>
```

두 가지 해결법이 있습니다. 이 select는 사용자가 직접 조작하는 게 아니라(실제 선택은 위의 커스텀 UI가 함), 검증만 트리거하는 용도이니 어느 쪽이든 됩니다.

방법 A — `readOnly`(값은 동기화하되 경고 제거):

```tsx
<select required value={value} readOnly aria-hidden tabIndex={-1} ...>
```

방법 B — 비제어(`defaultValue`):

```tsx
<select required defaultValue={value} aria-hidden tabIndex={-1} ...>
```

값이 바깥 상태와 계속 같이 움직여야 하면 A를, 마운트 시 초기값만 필요하면 B를 권합니다. 커스텀 UI가 진짜 값을 들고 있으니 보통 A로 두면 무난합니다.

## 4. [제안] robots.ts — 로그인 전용 경로 보강

`src/app/robots.ts`의 disallow에 로그인해야 의미 있는 경로를 추가합니다. 크롤러가 색인 가치 없는 페이지에 시간을 안 쓰게 됩니다.

```tsx
disallow: [
  "/admin/",
  "/api/",
  "/chat/",
  "/profile/",        // /profile/edit만이 아니라 프로필 전반
  "/favorites/",
  "/notifications/",
  "/myposts/",
  "/create/",
  "/payment/",
  "/go/",             // 실시간 'use client' 페이지, 크롤 대상 아님
],
```

실제 라우트 폴더 이름과 정확히 맞는지(예: `/create/`가 맞는지, `/posts/create` 같은 형태인지) 한 번 대조해서 경로를 맞춰주세요.

## 5. [제안] 상세 metadata fallback에 noindex

`src/app/(user)/usedgoods/[id]/metadata.ts:17`(rental 쪽 metadata.ts도 동일)의 데이터 없음 fallback에 색인 거부를 넣습니다.

지금:

```ts
if (!data) return { title: "게시글을 찾을 수 없습니다" };
```

바꾼 뒤:

```ts
if (!data) {
  return {
    title: "게시글을 찾을 수 없습니다",
    robots: { index: false },
  };
}
```

페이지가 `notFound()`로 404를 내니 대개 문제는 없지만, 만에 하나 200으로 새더라도 이 "찾을 수 없습니다" 페이지가 검색에 잡히지 않게 하는 보험입니다.
