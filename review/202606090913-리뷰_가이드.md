## 1. `users` 테이블 RLS — 개인정보 노출 막기 (최우선)

지금은 publishable 키로 누구나 전체 회원의 이메일·전화를 읽을 수 있는 상태입니다. Supabase 대시보드 → SQL Editor에서 정책을 추가하세요.

### 방법 A — "본인 행만 읽기" (가장 간단·안전)
```sql
alter table public.users enable row level security;

-- 로그인한 사용자가 자기 행만 select
create policy "users_select_own"
on public.users for select
to authenticated
using (auth_id = (select auth.uid()));
```

문제는 목록·상세에서 **판매자 이름/아바타**를 보여줘야 한다는 점입니다. 그때는 공개해도 되는 컬럼만 뷰로 빼서 그 뷰만 읽게 하세요.

### 방법 B — 공개 프로필 뷰 (이름·아바타만 노출)
```sql
create or replace view public.public_profiles as
  select id, name, avatar_url from public.users;

-- 뷰는 RLS가 기본 막히지 않으니, 원본 users는 본인만(방법 A) + 뷰는 읽기 허용
grant select on public.public_profiles to anon, authenticated;
```
그리고 목록/상세에서 판매자 정보를 join할 때 `users(*)` 대신 `public_profiles`를 쓰면 됩니다. `email`·`phone`은 절대 클라이언트로 내려가지 않게 됩니다.

> 확인: 정책 적용 후 `GET /rest/v1/users?select=*`를 publishable 키로 다시 호출해 **빈 배열이 오는지** 확인하세요.

## 2. `/chats`·`/jobs` 404 — 라우트 수정 + 상수화

### 당장 고치기
```ts

// src/app/(user)/usedgoods/[id]/_components/UsedGoodsDetail.tsx:181
```

`/jobs`는 두 가지 중 택1:
- 구인 **목록 페이지**를 만들 거라면 `src/app/(user)/jobs/page.tsx`를 만들고 `router.push("/jobs")` 유지.
- 아직 목록이 없다면 작성 후 상세나 홈으로: `router.push("/")` 또는 `router.push(`/job/${post.id}`)`(상세 라우트가 있으면).

### 근본 해결 — 라우트 상수

```ts

  chat: (id?: number) => (id ? `/chat/${id}` : "/chat"),
  rental: (id: number) => `/rental/${id}`,
  jobCreate: "/job/create",

// 사용
```

## 3. 작성 실패 시 정리(보상 트랜잭션) + 찜 롤백

### 3-1. 구인 작성: jobs 실패 시 posts 고아 정리
`CreateJobForm.tsx`의 jobs insert 실패 분기에 이미지 실패 때와 똑같이 post 삭제를 넣습니다.
```ts
const { error: jobError } = await supabase.from("jobs").insert({ ... });
if (jobError) {
  await supabase.from("posts").delete().eq("id", post.id); // ← 추가
  alert("공고 등록에 실패했습니다.");
  setIsSubmitting(false);
  return;
}
```
> 더 나아가면 이 "posts → 상세테이블" 2단계 저장은 서버에서 **RPC(트랜잭션)** 하나로 묶는 게 가장 안전합니다. 지금은 위 보상 로직으로 충분합니다.

### 3-2. 방렌트 찜: 롤백 추가 (useLikes처럼)
```ts
const handleLikeToggle = async (id: number) => {
  if (!isLoggedIn || currentUserId === null) { setShowLoginModal(true); return; }
  const target = items.find((i) => i.id === id);
  if (!target) return;
  const wasLiked = target.isLiked;

  setItems((prev) => prev.map((i) => i.id === id
    ? { ...i, isLiked: !wasLiked, likeCount: i.likeCount + (wasLiked ? -1 : 1) } : i));

  const { error } = wasLiked
    ? await supabase.from("common_likes").delete()
        .eq("user_id", currentUserId).eq("target_type", "post").eq("target_id", id)
    : await supabase.from("common_likes")
        .insert({ user_id: currentUserId, target_type: "post", target_id: id });

  if (error) {  // ← 실패 시 되돌리기
    setItems((prev) => prev.map((i) => i.id === id
      ? { ...i, isLiked: wasLiked, likeCount: i.likeCount + (wasLiked ? 1 : -1) } : i));
  }
};
```
