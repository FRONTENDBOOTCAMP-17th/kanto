## 1. [필수·보안] anon 키로 `users` 테이블이 통째로 읽히는 문제

### 지금 무슨 일이 벌어지나
publishable(anon) 키만 있으면 브라우저 콘솔에서든 curl로든 이렇게 전부 읽힙니다.

```bash
curl "$SUPABASE_URL/rest/v1/users?select=email,role,phone&role=eq.admin" \
  -H "apikey: <anon>" -H "Authorization: Bearer <anon>"
# → 관리자 이메일 목록이 그대로 나옴
```

RLS가 `users`에만 안 걸려 있어서 그렇습니다. 우리 서비스에서 `users`의 이메일·전화·role은 남이 볼 이유가 전혀 없는 값이니까요.

### 해결 — Supabase SQL 에디터에서 RLS를 본인 행으로 좁히기

```sql
-- 1) RLS 활성화 (이미 켜져 있으면 생략)
alter table public.users enable row level security;

-- 2) 기존의 느슨한 SELECT 정책이 있으면 먼저 제거
drop policy if exists "users are viewable by everyone" on public.users;

-- 3) 본인 행만 읽도록
create policy "users can read own row"
on public.users for select
to authenticated
using (auth_id = auth.uid());
```

이렇게 하면 로그인한 사용자는 자기 행만 읽습니다. 다만 다른 사람의 **이름·아바타**는 게시글 목록이나 채팅에서 화면에 보여야 하니, 그건 민감정보(email·phone·role)를 뺀 **공개 뷰**를 따로 만들어 그쪽만 열어 주면 됩니다.

```sql
create view public.public_profiles as
  select id, name, avatar_url from public.users;

grant select on public.public_profiles to anon, authenticated;
```

코드에서 "남의 프로필을 보여줄 때"는 `users`가 아니라 `public_profiles`를 조회하도록 바꿔 줍니다. 조인하던 곳(`posts ... users(name, avatar_url)`)도 이 뷰를 바라보게 하면 화면은 그대로 나오면서 이메일은 새어 나가지 않습니다.

> 배포 전에 `curl ".../rest/v1/users?select=email"`를 다시 한 번 찔러 보세요. `[]`나 401/권한 오류가 나오면 닫힌 겁니다.

## 2. [필수·보안] `bulkPostActions` / `bulkUserActions`에 관리자 인가 추가

`requireAdmin()` 헬퍼(`user.ts:48`)를 이미 잘 만들어 두셨는데, 다른 서버액션엔 다 붙어 있고 이 두 파일에만 빠졌습니다. 각 함수 맨 앞에 한 줄만 넣으면 됩니다.

```ts
// admin/posts/_actions/bulkPostActions.ts
export async function bulkDeletePosts(ids: string[]) {
  await requireAdmin();               // ← 이 한 줄. 관리자 아니면 여기서 throw
  const supabase = createAdminClient();
  // ...기존 로직
}
```

`bulkTogglePostStatus`·`bulkRestorePosts`와 `bulkUserActions.ts`의 함수들도 똑같이 첫 줄에 `await requireAdmin()`을 넣어 줍니다. `getCurrentAdminId()`는 id를 돌려줄 뿐 차단은 안 하니, 차단은 `requireAdmin()`에 맡기는 게 맞습니다.

## 3. [필수] 삭제글 상세 500 — `getUsedGoodsItem`에 삭제 필터

형제 함수엔 있는 필터가 이 함수에만 빠져 있습니다.

```ts
// services/usedGoods/usedGoods.ts  (getUsedGoodsItem)
const { data } = await supabase
  .from("used_goods")
  .select("*, posts(*, users(...))")
  .eq("id", id)
  .neq("posts.status", "deleted")   // ← getUsedGoodsDetail(:83)에 있는 것과 동일하게
  .maybeSingle();
```

`.neq`로 걸러도 조인 특성상 `posts`가 null로 올 수 있으니, 상세 페이지에서 한 번 더 막아 주면 안전합니다.

```tsx
// page.tsx
const data = await getUsedGoodsItem(id);
if (!data || !data.posts) notFound();   // posts 없으면(삭제/비공개) 404
```

이렇게 하면 삭제글은 500이 아니라 깔끔하게 404로 처리됩니다.

## 4. [필수] cleanup 라우트 fail-open → fail-closed

```ts
// api/posts/cleanup/route.ts
export async function GET(req: Request) {
  const secret = process.env.CRON_SECRET;
  const header = req.headers.get("authorization");

  // 시크릿이 없거나(=설정 누락) 일치하지 않으면 무조건 거부
  if (!secret || header !== `Bearer ${secret}`) {
    return new Response("Unauthorized", { status: 401 });
  }
  // ...삭제 로직
}
```

핵심은 `if (secret) { 검증 }`(있을 때만 검증)을 `if (!secret) 거부`(없으면 거부)로 뒤집는 겁니다. 설정이 빠진 상태가 "무방비"가 아니라 "차단"이 되도록 하자는 거죠.

## 5. [필수] 채팅 rental 오타

```ts
// api/chat/[id]/route.ts:61
- if (posts?.post_type === "rentals") {
+ if (posts?.post_type === "rental") {
```

한 글자입니다. 이것 때문에 렌탈 채팅에서 가격이 계속 null로 나왔습니다.

### 재발 방지 (선택) — 리터럴을 상수로
이런 오타를 아예 컴파일 타임에 잡으려면 post_type을 한 곳에 모아 두면 됩니다.

```ts
// lib/constants.ts
export const POST_TYPE = {
  used: "used_goods",
  rental: "rental",
  job: "jobs",
} as const;
export type PostType = (typeof POST_TYPE)[keyof typeof POST_TYPE];
```

이후 문자열 대신 `POST_TYPE.rental`을 쓰면 오타를 내는 순간 tsc가 빨간 줄을 그어 줍니다.

## 6. [제안] CI에 eslint 추가

`.github/workflows/ci.yml`의 tsc·build 스텝 옆에 한 줄 추가:

```yaml
      - name: Lint
        run: npx eslint .
```

지금 걸리는 9개 error 중 `PaymentCard.tsx:37`(렌더 중 `Date.now()`)와 `Chatbot.tsx:94`(렌더 중 ref 갱신)는 다시 렌더될 때 예상치 못한 값이 나올 수 있는 자리라, 단순 스타일 경고가 아닙니다. `Date.now()`는 effect 안이나 이벤트 핸들러로 옮기고 ref 갱신도 effect 안에서 하도록 바꾸면 룰이 풀립니다.
